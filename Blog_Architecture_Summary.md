# 📝 GitHub Pages 블로그 마이그레이션 및 인프라 요약

> **[Agent Context Handoff]**
> 이 파일은 새로운 채팅 세션에서 AI 에이전트가 블로그(Chirpy 테마)의 상태와 인프라, 그리고 **유저의 블로그 철학**을 파악하기 위한 블로그 전용 인수인계 문서입니다.

## 📌 유저 프로필 및 블로그 철학 (다음 AI 튜터를 위한 인수인계)
- **학습 성향**: 단순 코드 복붙을 혐오하며, 아키텍처의 인과관계와 근본 원리를 이해하는 것을 중시.
- **블로그 철학**: 업계 표준 용어(예: TL;DR)라도 본인이 이해하지 못하거나 어색하면 과감히 배제하고 직관적인 표기법(📌 3줄 요약)을 선호.
- **작성 원칙**: 임의의 가짜 수치(Fake Metrics) 창조 절대 금지, 모든 글은 터미널에서 직접 부딪히고 해결한 100% 팩트 기반. 회고 작성 시 딱딱한 'Q&A' 대신 '💡 회고 및 Lessons Learned' 포맷 도입.

## 1. 아키텍처 및 CI/CD 상태
- **테마:** Jekyll Chirpy Theme (정적 사이트 생성기)
- **호스팅/FinOps:** GitHub Pages 서버리스 운영을 통해 EC2 호스팅(월 약 2만 원) 대비 유지비 0원 달성.
- **CI/CD 파이프라인:** GitHub Actions (`.github/workflows/deploy.yml`)
  - 푸시(Push) 발생 시 마크다운 문법 검사(`markdownlint`) 및 빌드 검증을 거쳐 에러가 없을 때만 배포됨(코드 붕괴를 막는 안전망).
  - *(참고: Slack Webhook은 추가하려다 Secret 의존성 이슈로 제거함)*

## 2. SEO (검색 엔진 최적화) 트러블슈팅 완료
- **구글/네이버 연동:** 서치어드바이저 및 구글 애널리틱스 연동 완료.
- **404 TTL 캐싱 인지:** 네이버 로봇 서버 측의 404 캐싱 쿨타임을 네트워크 엔지니어의 시각으로 분석하고 대기하여 해결.
- **Silent Failure 방어:** Chirpy 테마 버전업으로 인한 `analytics: google: id:` YAML 계층구조 변경을, 로컬 `curl` 렌더링 팩트 체크를 통해 디버깅.

## 3. 데이터 마이그레이션 및 포트폴리오 현황
- **데이터:** Python 스크립트로 Velog 162개 글 카테고리 자동화 및 Syntax Highlighting 강제 복구 완료.
- **CV:** `_tabs/about.md`를 FinOps/DevOps 역량이 돋보이는 시니어급 이력서 포맷으로 리팩토링.
- **댓글망 (Giscus):** GitHub Discussions 기반 서버리스 댓글 연동 (title 매핑으로 인코딩 버그 방어).

## 🎯 향후 유지보수 방향
- 타 프로젝트(Cover Challenge 등)에서 겪은 백엔드 트러블슈팅 로그를 이 블로그 세션으로 들고 와서, 고퀄리티 마크다운 아티팩트로 렌더링하고 `git push`를 통해 CI/CD로 자동 배포.
