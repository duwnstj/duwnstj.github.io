## 개요
`for`문은 프로그래밍에서 가장 기본적인 반복문 중 하나입니다. 그러나 이중, 삼중, 또는 더 깊은 중첩 `for`문에서 **수행 횟수**를 정확히 계산하는 일은 까다로울 수 있습니다. 특히, 조건이 붙는 for문에서는 **수행 횟수와 시간 복잡도**를 계산하기 위해 **조합 공식**을 활용해야 하는 경우가 많습니다.

***
## 1. 시간복잡도란?
시간복잡도는 **입력 크기 𝑛**에 따라 알고리즘의 실행 시간이 어떻게 변하는지를 나타냅니다.
특히, 시간복잡도는 **최악의 경우**를 기준으로 계산합니다.

**왜 최악의 경우를 기준으로 하나요?**
1. **안정성과 신뢰성**
- 최악의 상황에서도 알고리즘이 작동 가능한지 확인할 수 있습니다.
- 예를 들어 정렬 알고리즘이 $$O(n^2)$$인 경우 n이 큰 데이터에서도 성능이 보장되는지 알 수 있습니다.
2. **일관된 비교**
- 모든 알고리즘을 동일한 기준(최악의 경우)으로 비교하면 성능 평가가 간단해집니다
3. **실질적인 의미**
- 평균이나 최선의 경우는 데이터의 분포에 따라 달라질 수 있지만, 최악의 경우는 항상 고정적입니다.
***

## 2. 이중 for문의 수행 횟수와 시간복잡도
다음과 같은 이중 `for`문을 살펴봅시다

{% highlight java %}
for (int i = 1; i <= n; i++) {
    for (int j = i + 1; j <= n; j++) {
        // 수행 코드
    }
}

{% endhighlight %}
**수행 횟수 분석**
- i는 1부터 n−1까지 반복합니다.
- j는 i+1부터 n까지 반복합니다.
- 조건 i<j를 만족하는 모든 i,j 조합을 탐색합니다.

**총 수행 횟수**
이중 for문의 수행 횟수는 **2개를 선택하는 조합의 수**와 동일합니다
![](https://velog.velcdn.com/images/duwnstj12/post/96a77d46-a180-4de7-bd2c-ec479c1d8c68/image.png)

**가정 : n=5일 때**
i = 1~5까지 반복 1,2,3,4,5 [5]
j = i+1~n까지 반복 2,3,4,5 [4]
2! : (1,2)(2,1)     [2]가지 경우의 수 
즉 `5*4/2`로 나눈 값인 10번의 수행횟수가 나오게 됩니다.
- 공식에 대입해보면 `n(n-1)/2`라는 공식이 나오게 됩니다.

**시간 복잡도**
![](https://velog.velcdn.com/images/duwnstj12/post/ece1cc22-5db2-48cb-99e8-dfb1051154e8/image.png)
- $$O(n^2)$$이 나오는 이유는 최악의 경우로 나타내기 때문입니다.
- 최악의 경우로는 **i~n,j~n** 이 경우가 있습니다.
***
## 3. 삼중 for문의 수행횟수와 시간복잡도
삼중 for문에서는 다음과 같은 조건이 있습니다

{% highlight java %}
for (int i = 1; i <= n; i++) {
    for (int j = i + 1; j <= n; j++) {
        for (int k = j + 1; k <= n; k++) {
            // 수행 코드
        }
    }
}

{% endhighlight %}
**수행 횟수 분석**
- i<j<k 조건을 만족하는 모든 i,j,k 조합을 탐색합니다.

**공식: 조합으로 계산**
총 수행 횟수는 **3개를 선택하는 조합의 수**와 동일합니다
![](https://velog.velcdn.com/images/duwnstj12/post/567a4af0-7646-4a95-9266-239dda757ca8/image.png)

**가정 : n=5일 때**
i = 1~5까지 반복 1,2,3,4,5 [5]
j = 2~5까지 반복 2,3,4,5 [4]
k = 3~5까지 반복 3,4,5[3]
3! = `3*2*1` (1,2)(1,3)(2,1)(2,3)(3,1)(3,2) [6]
즉 `5*4*3/6` = 10번의 수행횟수가 나오게됩니다.
- 공식에 대입해보면 `n(n-1)(n-2)/6`공식이 나오게 됩니다.

**시간복잡도**
![](https://velog.velcdn.com/images/duwnstj12/post/ef2f642f-e066-40c1-ac36-c161be057af8/image.png)
- 최악의 경우는 i,j,k 모두 n까지 반복할 때 즉 $$n^3$$ 일때입니다.
