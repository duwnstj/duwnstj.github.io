API를 개발 중 Git에서 예상치 못한 브랜치가 생성되면서 코드 손실 문제가 발생했다. 커밋을 완료한 후 푸시하려 할 때 , 갑자기 알 수 없는 브랜치가 생성되어 그 브랜치로 푸시를 시도했다.
그래서 푸시를 취소한 후 코드를 다시 확인하려 했으나, 코드가 사라진것을 발견했다.
이를 해결 하기 위해 `git reflog` 명령어를 사용해 코드를 복구했다.

### 1. 문제 발생 
`User` 엔티티 저장 기능을 구현한 후 `master` 브랜치에서 커밋을 진행했다. 그러나 커밋 후 푸시를 시도하려는 순간, 예상치 못한 브랜치가 자동으로 생성되었고, Intellij가 그 브랜치로 푸시를 시도했다.
당황한 상태에서 푸시를 취소했으나, 그 결과로 작업하던 코드가 사라진 상황이 발생했다.

### 2. 문제 원인
이 문제는 Intellij IDEA와 Git 설정의 몇가지 요인으로 인해 발생할 수 있다.

1. **자동 브랜치 생성 기능 관련 설정**
Intellij IDEA는 리베이스나 머지 작업중 충돌이 발생할 경우 임시 브랜치를 생성할 수 있는데, 이러한 자동 브랜치 생성 기능이 예상치 못한 시점에 작동해서 새로운 브랜치가 생성되었을 가능성이 있다.

2. **로컬 브랜치와 원격 브랜치 간의 동기화 문제**
로컬 브랜치와 원격 브랜치 간의 불일치나 충돌이 발생하면, Intellij가 이를 해결하기 위해서 새로운 브랜치를 생성할 수 있다. 이런 경우 , 로컬 브랜치가 최신 원격 브랜치와 제대로 동기화되어 있는지 확인해야한다.

### 3. 문제 해결 과정
문제를 해결하기 위해 다음과 같이 해결했다.
1. `git reflog` **명령어 실행**
`git reflog` 명령어를 사용하여 최근 브랜치 이동과 커밋 기록을 확인했다 아래 이미지는 이 과정에서 사용한 명령어와 결과이다.
![](https://velog.velcdn.com/images/duwnstj12/post/915203d5-a440-4191-9ebd-e5cab592fff7/image.png)

2. **해당 커밋 찾기**
`reflog` 결과에서 잘못된 브랜치에 커밋된 내역을 확인하고, 해당 커밋 해시를 찾았다. 이를 바탕으로 복구를 진행했다.

3. **복구 브랜치 생성**
아래 이미지는 커밋 해시를 기반으로 새로운 브랜치를 생성하여 안전하게 코드를 복구하는 과정이다. `git chexkout -b recovered <커밋 해시>` 명령어를 사용하여 `recovered`라는 새로운 브랜치를 만들었다.
![](https://velog.velcdn.com/images/duwnstj12/post/4a18d638-3688-4c08-aa08-2148c1e34408/image.png)

4. **Git PullRequest 사용**
PullRequest를 사용하여 master 브랜치로 병합을 한 후 다시 master 브랜치에 푸시를 함으로써 문제가 해결됐다.
