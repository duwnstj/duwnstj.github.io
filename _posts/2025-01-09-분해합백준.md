## 문제 설명
이 문제는 완전 탐색(Brute Force) 를 이용하여 해결하는 문제입니다.
주어진 자연수 N에대해 **생성자**를 구하는 문제입니다. **생성자**란 아래의 조건을 만족하는 자연수 M입니다.

$$N = M+(M의 각 자리수의 합)$$
-  생성자 M중 가장 작은 값을 출력해야합니다.
- 만약 생성자가 존재하지 않는다면 0을 출력합니다.
***

 ## 풀이 코드
 ```java
 import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        /**
         * N  = N + N을 이루는 각 자리수의 합
         * N = M+M의 분해합 == 생성자 출력
         * N != N+N의 분해합 == 0 출력
         */
        Scanner sc = new Scanner(System.in);
        int N = sc.nextInt();
        int result = 0; // 결과 값 
        
        // M이 주어지지 않아서 1~N까지 탐색
        for (int M = 1; M <= N; M++) {
            int number = M;
            int sum = 0;  // 각 자리수 합

            while (number > 0) {
                sum += number % 10;
                number /= 10;
            }

            if (sum + M == N) {
                result = M;
                break;
            }
        }
        System.out.println(result);

    }
}

 ```
 ***
### 풀이 과정
1. **생성자 탐색 범위**
- M의 범위는 1부터 N까지입니다.
2. **분해합 계산**
- M의 분해합은 M+M의 각 자리수의 합으로 계산됩니다.
- 예를 들어 M = 198일 때

$$분해합 = 198+1+9+8=216$$

3. **가장 작은 생성자 찾기**
- 탐색 중 M의 분해합이 N과 같으면 M을 저장하고 탐색을 종료합니다.
4. **결과 처리**
- 생성자가 발견되면 출력하고 없으면 0을 출력합니다.
***
## 시간 복잡도 
- N까지 반복하므로 시간 복잡도는 **O(N*d)**입니다.
- d : 각 숫자의 자리수

- **최악의 경우**
    - N = 10,00000이라면 최대 60,00000번의 연산이 필요하게됩니다.
