## 문제 설명
주어진 알고리즘은 다음과 같습니다.
{% highlight java %}
MenOfPassion(A[], n) {
    sum <- 0;
    for i <- 1 to n - 1
        for j <- i + 1 to n
            sum <- sum + A[i] × A[j]; # 코드1
    return sum;
}

{% endhighlight %}
이 알고리즘에서 중요한 부분은 이중 for문입니다. 
첫번째 반복문은 `i=1`부터 `n-1`까지 반복하고 두번째 반복문은 `j=i+1`부터 `n`까지 반복합니다. 
즉 `i` 값에 따라 `j` 값의 범위가 달라지기 때문에 전체 반복횟수를 구하는 것이 핵심입니다.
***

## 알고리즘 실행 예시
`n=7`일경우 반복문의 동작은 다음과 같습니다.

{% highlight text %}
i = 1일 때, j는 [2, 3, 4, 5, 6, 7] (6번 반복)
i = 2일 때, j는 [3, 4, 5, 6, 7] (5번 반복)
i = 3일 때, j는 [4, 5, 6, 7] (4번 반복)
i = 4일 때, j는 [5, 6, 7] (3번 반복)
i = 5일 때, j는 [6, 7] (2번 반복)
i = 6일 때, j는 [7] (1번 반복)
{% endhighlight %}
따라서 전체 반복 횟수는 `6+5+4+3+2+1 =21`번이 됩니다.
***

## 수행 횟수 계산
이 알고리즘의 전체 반복 횟수는 `1 + 2 + 3 + ... + (n - 1)`의 합으로, 이는 등차수열의 합입니다. 등차수열의 합 공식에 의하면, 해당 값은 다음과 같습니다
![](https://velog.velcdn.com/images/duwnstj12/post/482cfb85-87a3-4422-bf6a-b336c96031d3/image.png)
따라서 코드1의 수행 횟수는 n(n - 1) / 2로 계산됩니다.
***
## 시간 복잡도 분석
위에서 구한 수행 횟수는 `n(n - 1) / 2`입니다. 이 값은 큰 입력 값에 대해 시간 복잡도를 분석할 때 중요한 지표가 됩니다.
이 수행 횟수는 $$O(n^2)$$에 해당합니다. 왜냐하면 `n`이 커질수록 수행 횟수는 $$n^2$$에 비례하여 증가하게 되기 때문입니다. 이를 통해 주어진 알고리즘의 시간 복잡도는 $$O(n^2)$$로 판별할 수 있습니다.
따라서 주어진 알고리즘은 입력 크기 `n`이 커짐에 따라 시간이 급격히 늘어나는 **이차 시간 복잡도**를 가지게 됩니다.
***
## 다항식의 최고차항 차수
문제에서 수행 횟수를 다항식으로 나타낼 때 최고차항의 차수를 구하는 것이 요구됩니다. 
수행 횟수 n(n - 1) / 2를 다항식 형태로 나타내면 다음과 같습니다
![](https://velog.velcdn.com/images/duwnstj12/post/6222efaa-9fad-4986-b945-9ba078469167/image.png)

여기서 $$n^2$$이 가장 높은 차항이므로, 수행 횟수의 최고차항 차수는 `2`입니다. 따라서 이 알고리즘의 최고차항 차수는 `2`입니다.
***

## 해결 코드
{% highlight java %}
import java.io.*;

public class Main {
    public static void main(String[] args) throws IOException {
      

        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
        Long n = Long.parseLong(br.readLine());
        // 코드1의 수행 횟수 출력
        bw.write(n * (n - 1) / 2+"\n");

        // 수행 횟수의 최고차항 차수는 2이다 (n^2의 차수는 2)
        bw.write(2+"\n");
        bw.flush();

    }
}

{% endhighlight %}
*** 
## 주의할 점
- `n`의 범위가 `n(1 ≤ n ≤ 500,000)`이기때문에 `int` 타입으로는 범위가 초과되어 `Overflow`가 발생하게 됩니다. 
- 따라서 `Long` 타입으로 `n`을 처리해주어야합니다.
- `Long.parseLong()`을 사용하여 `n` 값을 입력받아 문제를 해결했습니다.
