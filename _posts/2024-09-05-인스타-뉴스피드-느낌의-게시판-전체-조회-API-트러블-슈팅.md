## 겪은 문제 상황
이 API를 만들 때 먼저 한글로 어떻게 값을 가져와야할까하는 생각을 적어봤었다.
```text
유저 뉴스피드 전체 조회 API
조건 : 팔로잉을 한 사람들의 게시물과 내가 작성한 게시물이 최신순으로 내림차순 정렬이 되야한다.

문제점 : 팔로잉을 한 사람의 게시물을 어떻게 가져올것인가?  board와 follow 엔티티를 어떻게 연결해야하나
```
처음에  팔로우 기능이 없었기 때문에 jwt를 통해서 인증을 하고 나서 userId를 가져와 내 게시물들을 전체 조회하는 부분을 잘 되었었다.

## 기존 유저 게시물 전체 조회 비즈니스 로직 코드
```java
//유저 게시물 전체 조회
    public Page<GetDetailResponseDto> getUsersBoards(int page, int size, AuthUser authuser) {
        Pageable pageable = PageRequest.of(page - 1, size);
        User user = findUser(authuser.getId());
        Long userId = user.getId();

        List<Follow> followList = followRepository.findByFollowingId(userId);


        List<Long > userIds = followList.stream().map(follow -> follow.getFollowing.getId())
        .collect(Collectors.toList());
        userIds.add(userId);



        Page<Board> boards = boardRepository.findByUserIdInOrderByCreationDateDesc(userIds, pageable);
        return boards.map(newboard -> new GetDetailResponseDto(
                user.getUserName(),
                newboard.getContent(),
                newboard.getTitle(),
                newboard.getCreationDate(),
                newboard.getModifiedDate())
        );

    }
    
    // userId에 해당하는 유저 객체 찾기
    private User findUser(Long userId) {

        return userRepository.findById(userId)
                .orElseThrow(() -> new ResourceNotFoundException("user를 찾을 수 없습니다."));
    }
```
### 코드 설명 
**조건** : page가 처리되고 그 가져온 게시물들이 생성일을 기준으로 내림차순 정렬이 되어야했다.
따라서 Page 클래스를 사용해 기능을 구현을 했고 코드를 간단하게 설명해보겠다.

1. `userId`에 해당하는 유저 객체를 찾는 메서드가 중복이 많이 되어서 `findUser()`메서드로 만들어주었다. 이렇게 하여 AuthUser 클래스를 통해 받은 `userId`의 유저 정보를 가져온다.
2. `findByFollowingId` 쿼리 메서드를 사용하여 `userId` 즉 , 로그인 한 사람이 팔로잉을한 user의 정보를 followlist에 리스트로 저장해준다.
3. followlist에서 팔로우한 user의 Id를 추출하여 `userIds` 리스트에 담아주고 로그인을 한 사람의 userId도 같이 .add()메서드를 사용하여 저장해준다.
    - 이렇게 하면 `userIds`에 팔로우한 사람의 userId,로그인을 한사람의 userId가 저장이 된다.
4. 리턴할 board 객체가 페이징이 되어야하기 때문에 Page 클래스를 사용하여 페이징 처리를 해주었고 `findByUserIdInOrderByCreationDateDesc` 이 쿼리메시지를 사용하여 userId의 리스트들의 게시물들을 생성일자 기준으로 내림차순 정렬을 해주었다. 
5. board 엔티티의 정보를 직접 클라이언트에 보내주면 보안적인 측면에서 안좋기 때문에 ResponseDto 객체로 stream.map()메서드를 사용하여 변환해주어 컨트롤러에 리턴을 해주었다.

## 문제점
코드를 짠 후 PostMan을 사용해 요청을 보내봤더니 내 생각과는 다르게 로그인한 유저의 게시물만 가져오는 문제가 생겼다. 그래서 디버깅을 비즈니스 로직에서 해보았다.

### 디버깅 
총 3번의 브레이킹포인트를 정해준 후 디버깅을 하였다.
![](https://velog.velcdn.com/images/duwnstj12/post/743eb3f9-40ad-4ad1-b07f-47dae6b4fdfb/image.png)
1. 첫번째 브레이킹 포인트는 `followList`에 팔로우한 사람들의 user정보가 잘 담기는지 확인하기 위해 걸어주었다.
2. 두번째 브레이킹 포인트는 `userIds`에 followList에서 추출한 userId 값이 여러개로 잘 담기는지 확인해주었다.
3. 세번째 브레이킹 포인트는 boards 객체에 가져올 게시물들이 잘 담겨오는지 확인하기 위해 걸어주었다.

### 디버깅 문제점
디버깅을 하면서 문제점을 찾을 수 있었다. 그 문제는 두번째 브레이킹 포인트에서 일어났다. userIds에 로그인한 사람의 userId인 5가 3개가 담기는것을 확인할 수 있었다.
![](https://velog.velcdn.com/images/duwnstj12/post/2f7bc1c8-582d-47c7-8efc-50be2f8f6c7a/image.png)
이미지에서 보다시피 5가 3개가 찍혀있던것이였다... 그러니 게시물을가져올때 로그인한 유저의 게시물만 가져왔던것이였다 

### 코드의 문제점 
그래서 왜 이런 결과가 나왔나 하고 Follow 데이터베이스를 확인해봤다.
![](https://velog.velcdn.com/images/duwnstj12/post/0d38bc20-a80b-4386-b9cd-550a3b599ca7/image.png)
확인을 해보고 다시 코드를 보니 
`userIds = followList.stream().map(follow -> follow.getFollowing.getId())` 이코드에서 문제점을 찾게 되었다. 바로 getFollowing이 문제였던 것이다.
Follow Entity를 확인을 해보면 이렇게 구성이 되어있다.

```java
package com.sparta.headlinehub.entity;

import jakarta.persistence.*;
import lombok.Getter;
import lombok.NoArgsConstructor;

@Entity
@Table(name = "follow")
@Getter
@NoArgsConstructor
public class Follow extends Timestamped{
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @ManyToOne
    @JoinColumn(name = "following_id")
    private User following;

    @ManyToOne
    @JoinColumn(name = "follower_id")
    private User follower;

    public Follow(User my, User user) {
        this.following = my;
        this.follower = user;
    }
}
```
이 코드를 확인해보면 following이 아닌 follower의 리스트를 가져와야하는데 following의 리스트를 가져오니 로그인한 사람의 userId가 userIds에 저장이 되는것이였다..

## 해결 방법
문제가 생긴 코드를 `userIds = followList.stream().map(follow -> follow.getfollower.getId())`로 수정하여 문제를 해결했다.


기술블로그를 작성하다보니 코드의 문제점을 하나더 발견하였다. 그부분은 팔로우한 사람의 게시물과 내 게시물들을 잘 가져왔는데 user의 네임이 로그인한 계정인 `홍길동` 이름으로 전부 나오는 문제가 있었다. 

```java
GetDetailResponseDto(
                user.getUserName(),
                newboard.getContent(),
                newboard.getTitle(),
                newboard.getCreationDate(),
                newboard.getModifiedDate())
        );
```
이 코드를 확인해보면 user.getUserName()을 가져오는데 이 코드는 결국 로그인한 유저의 이름을 가져오는 코드이다. 따라서 아래와 같이 코드를 변경해주었다.
`newboard.getUser().getUserName()` 이 코드는 게시물을 작성한 사람의 이름을 가져오는 코드이다.
