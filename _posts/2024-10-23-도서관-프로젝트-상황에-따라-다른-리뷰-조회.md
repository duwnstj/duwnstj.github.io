## 개요
- 처음에는 리뷰 조회를 위해 여러 개의 API를 만들어, 로그인한 사용자가 작성한 리뷰만을 조회하는 로직을 먼저 구현했습니다

- 하지만 QueryDSL을 사용하면 다양한 조건을 동적으로 처리할 수 있을 것 같다는 생각이 들었습니다. 이로인해 로그인한 사용자의 리뷰를 조회하는 로직과 여러 조건을 처리하는 동적 조회 로직을 하나의 API로 구현할 수 있지 않을까 고민했습니다.

- 그러나 `Spring Security`를 통해 인증 및 인가 처리가 이미 완료된 상황에서는 로그인한 사용자의 정보가 `null`이 될수 없다는 점을 고려해야했습니다.

- 결과적으로 로그인한 사용자의 리뷰만을 조회하는 API와 QueryDSL을 사용하여 다양한 조건을 처리하는 API를 별도로 구현하게 되었습니다.

## 로그인한 사용자의 리뷰 조회(Controller)

```java
 /**
     * 로그인한 유저가 작성한 리뷰 전체 조회
     *
     * @param userDetails : 로그인한 유저정보
     * @return : 리뷰 정보
     */
    @GetMapping("/reviews/user")
    public ResponseEntity<Page<ReviewsGetResponse>> findAllByUserWriten(
            @RequestParam(defaultValue = "1") int page,
            @RequestParam(defaultValue = "10") int size,
            @AuthenticationPrincipal UserDetailsImpl userDetails
    ) {
        return ResponseEntity.ok(reviewService.findAllUserWriten(page, size, userDetails));
    }
```
- 이 코드는 페이징 처리를 하기 위해 `page와 size`를 `RequestParam`으로 받아오는것을 볼수 있습니다.

## 로그인한 사용자의 리뷰 조회(Service)
```java
//로그인한 유저가 작성한 리뷰 조회
    public Page<ReviewsGetResponse> findAllUserWriten(int page
            , int size
            , @AuthenticationPrincipal UserDetailsImpl userDetails) {
        Pageable pageable = PageRequest.of(page - 1, size);
        Page<Review> reviewList = reviewRepository.findAllUserWriten(pageable, userDetails.getUser().getId());

        return reviewList.map(list -> new ReviewsGetResponse(
                list.getBook().getId(),
                list.getId(),
                list.getReviewStar(),
                list.getReviewTitle(),
                list.getReviewDescription(),
                list.getCreatedAt(),
                list.getModifiedAt()
        ));

    }
```
- 여기서는 `Pageable` 인터페이스를 사용하여 페이지 정보를 전달하고, 이를 통해 `Page<Review>` 타입의 페이징된 리뷰 목록을 reviewList로 받습니다.
- 마지막으로 페이징 된 `reviewList`를 스트림의 `.map()`메서드를 사용해`ReviewsGetresponse`객체로 변환한 후 클라이언트에게 응답합니다. 

## 로그인한사람의 리뷰 조회(Repository)
```java
package com.example.library_management.domain.review.repository;


import com.example.library_management.domain.review.entity.Review;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.Pageable;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;

public interface ReviewRepository extends JpaRepository<Review, Long>, ReviewQueryDslRepository {


    @Query("select r from Review r where r.user.id =:userId")
    Page<Review> findAllUserWriten(Pageable pageable, @Param("userId") Long userId);
}
```
- `@Query` 애노테이션을 사용하여 JPQL로 쿼리문을 작성해주었습니다.
    - @Query("select r from Review r where r.user.id =:userId")
    - 이 쿼리문은 `userId`가 일치하는 사용자가 작성한 리뷰를 조회하는 기능을 수행합니다.

## QueryDSL을 통한 다양한 조건 조회


### build.gradle 설정
-  QueryDSL을 사용하기 위해 `build.gradle`파일에 필요한 의존성을 추가했습니다.
```java
 // querydsl
    implementation 'com.querydsl:querydsl-jpa:5.0.0:jakarta'
    annotationProcessor "com.querydsl:querydsl-apt:5.0.0:jakarta"
    annotationProcessor "jakarta.annotation:jakarta.annotation-api"
    annotationProcessor "jakarta.persistence:jakarta.persistence-api"

```
### QueryDSL설정(JPAQueryFactory 빈 등록)
- QueryDSL은 Q 클래스를 통해 엔티티에 접근하므로, `JPAQueryFactory`를 빈으로 등록해 사용해야 합니다. 이를 위해 `config` 패키지에 `QueryDslConfig` 클래스를 추가하고, `JPAQueryFactory`를 빈으로 등록했습니다.

```java
package com.example.library_management.global.config;

import com.querydsl.jpa.JPQLTemplates;
import com.querydsl.jpa.impl.JPAQueryFactory;
import jakarta.persistence.EntityManager;
import jakarta.persistence.PersistenceContext;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
class QueryDslConfig {

    @PersistenceContext
    private EntityManager em;

    @Bean
    public JPAQueryFactory jpaQueryFactory(){
        return new JPAQueryFactory(JPQLTemplates.DEFAULT,em);
    }
}

```
## ReviewRepository에서 인터페이스 다중상속

- 그 후 ReviewRepository에서 `ReviewQueryDslRepository`를 추가로 상속을 받아 QueryDSL을 사용할 수 있도록 했습니다.
- 자바에서 클래스는 다중 상속이 안되지만 인터페이스는 다중 상속이 가능합니다.
```java
package com.example.library_management.domain.review.repository;


import com.example.library_management.domain.review.entity.Review;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.Pageable;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;

public interface ReviewRepository extends JpaRepository<Review, Long>, ReviewQueryDslRepository {


    @Query("select r from Review r where r.user.id =:userId")
    Page<Review> findAllUserWriten(Pageable pageable, @Param("userId") Long userId);
}

```
### ReviewQueryDslRepository 구현 클래스 생성

- 그 후 `ReviewQueryDslRepository`에서 선언한 추상메서드를 구현하는 Impl 클래스를 만들어주었습니다.
- 이 때 주의할 점은 `@Repository` 애노테이션을 꼭 붙여주어야한다는 점입니다. 이 애노테이션을 붙여줘야 스프링 부트가 빈으로 등록을 해줍니다.


### ReviewQueryDslRepositoryImpl 클래스
```java
package com.example.library_management.domain.review.repository;

import com.example.library_management.domain.review.dto.response.ReviewsGetResponse;
import com.querydsl.core.types.Projections;
import com.querydsl.core.types.dsl.BooleanExpression;
import com.querydsl.core.types.dsl.Wildcard;
import com.querydsl.jpa.impl.JPAQueryFactory;
import lombok.RequiredArgsConstructor;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.PageImpl;
import org.springframework.data.domain.Pageable;
import org.springframework.stereotype.Repository;

import java.util.List;

import static com.example.library_management.domain.review.entity.QReview.review;

@Repository
@RequiredArgsConstructor
public class ReviewQueryDslRepositoryImpl implements ReviewQueryDslRepository {
    private final JPAQueryFactory jpaQueryFactory;


    @Override
    public Page<ReviewsGetResponse> findAllByMultipleConditions(

            Pageable pageable,
            Long bookId,
            Integer reviewStar) {
        // 페이징 된 데이터 가져오기
        List<ReviewsGetResponse> results = jpaQueryFactory
                .select(
                        Projections.constructor(
                                ReviewsGetResponse.class,
                                review.book.id,
                                review.id,
                                review.reviewStar,
                                review.reviewTitle,
                                review.reviewDescription,
                                review.createdAt,
                                review.modifiedAt

                        )
                )
                .from(review)
                .where(
                        bookIdAndReviewStar(bookId, reviewStar)

                ).offset(pageable.getOffset())
                .limit(pageable.getPageSize())
                .orderBy(review.createdAt.desc())
                .fetch();

        // 전체 페이지 데이터 개수 구하기
        Long totalCount = jpaQueryFactory
                .select(Wildcard.count)
                .from(review)
                .where(bookIdAndReviewStar(bookId, reviewStar)
                ).fetchOne();

        return new PageImpl<>(results, pageable, totalCount);

    }

    //해당하는 책과 리뷰 별점을 같이 조회하고 싶을때
    private BooleanExpression bookIdAndReviewStar(Long bookId, Integer reviewStar) {

        if (bookId != null && reviewStar != null) {
            return review.book.id.eq(bookId).and(review.reviewStar.eq(reviewStar));
        } else if (bookId != null) {
            return review.book.id.eq(bookId);
        }else if(reviewStar !=null){
            return review.reviewStar.eq(reviewStar);
        }
        return null;


    }
}
```
- 빈으로 등록한 `JPAQueryFactory`를 필드로 주입받아 사용합니다.

```java
.select(
                        Projections.constructor(
                                ReviewsGetResponse.class,
                                review.book.id,
                                review.id,
                                review.reviewStar,
                                review.reviewTitle,
                                review.reviewDescription,
                                review.createdAt,
                                review.modifiedAt

                        )
```
- 위 코드는 조회된 데이터를 `ReviewsGetResponse` 객체로 변환하여 필요한 정보만 담아 리턴합니다. QueryDSL에서는 `review`라는 Q 클래스를 사용하여 엔티티에 접근할 수 있습니다.

```java
// 페이징 된 데이터 가져오기
        List<ReviewsGetResponse> results = jpaQueryFactory
                .select(
                        Projections.constructor(
                                ReviewsGetResponse.class,
                                review.book.id,

                                review.reviewStar,

                                review.reviewDescription,

```
- 이 코드는 페이징된 데이터를 가져오며, `.where()` 구문에서 사용된 `bookIdAndReviewStar(bookId, reviewStar)` 메서드를 통해 동적 쿼리가 실행됩니다. 이 코드가 여러 조건을 처리하는 동적 쿼리의 핵심입니다.

```java
// 책 ID와 리뷰 별점을 기준으로 조건 조회
    private BooleanExpression bookIdAndReviewStar(Long bookId, Integer reviewStar) {
        if (bookId != null && reviewStar != null) {
            return review.book.id.eq(bookId).and(review.reviewStar.eq(reviewStar));
        } else if (bookId != null) {
            return review.book.id.eq(bookId);
        } else if (reviewStar != null) {
            return review.reviewStar.eq(reviewStar);
        }
        return null;
    }
```
- 이 메서드는 `BooleanExpression`을 사용하여 `true 또는 false` 값을 반환하며, 조건이 없을 경우 `null`을 반환합니다. QueryDSL은 `null`을 무시하여 해당 조건을 포함하지 않은 쿼리를 실행합니다. 

```java
).offset(pageable.getOffset())
                .limit(pageable.getPageSize())
                .orderBy(review.createdAt.desc())
                .fetch();
```
- 이 코드는 `.offset()`과 `.limit()` 메서드를 통해 페이징 처리를 하고, `review.createdAt.desc()`로 생성 날짜 기준 내림차순 정렬을 적용합니다. 마지막으로 `.fetch()` 메서드를 통해 리스트를 조회하여 페이징된 데이터를 가져옵니다.

```java
// 전체 데이터 개수 조회
        Long totalCount = jpaQueryFactory
                .select(Wildcard.count)
                .from(review)
                .where(bookIdAndReviewStar(bookId, reviewStar))
                .fetchOne();

        return new PageImpl<>(results, pageable, totalCount);
    }
```
- 페이징된 데이터의 총 개수를 조회하기 위해 `Wildcard.count`를 사용하여 전체 레코드 수를 가져옵니다. 이후 `PageImpl` 객체를 통해 데이터, 페이징 정보, 그리고 전체 데이터 개수를 함께 반환합니다. 이를 통해 QueryDSL을 사용하여 다양한 조건을 처리할 수 있습니다
