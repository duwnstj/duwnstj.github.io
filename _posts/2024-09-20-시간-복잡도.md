## 어떤 알고리즘으로 골라야할까?
알고리즘 선택을 할 때 가장 중요한 기준은 **시간 복잡도**이다. 알고리즘을 잘못 선택하면 , 아무리 코드가 깔끔하고 효율적이어도 처리 시간이 지나치게 오래 걸릴 수 있다. 
> 코딩 테스트의 핵심은 문제마다 주어진 시간 복잡도를 고려해 적절한 알고리즘을 선택하는 것이다.

시간 복잡도를 고려하지 않고 문제를 풀면 코드는 동작하지만, **시간초과**로 인해 문제를 완전히 해결하지 못할 수 있다. 따라서 알고리즘 선택시 시간 복잡도를 반드시 고려해야한다.

## 시간 복잡도 표기법 
알고리즘의 **시간 복잡도**는 문제를 해결하는데 필요한 **연산 횟수**를 나타낸다. 일반적으로 **1억번의 연산**이 약1초에 해당한다고 가정하고 알고리즘의 성능을 예측할 수 있다. 이를 통해 주어진 시간 내에 문제를 해결할 수 있는지를 판단한다. 
### 시간 복잡도의 주요 표기법
시간 복잡도는 세가지 표기법으로 구분된다. 각 표기번은 최선, 보통,그리고 최악의 경우에 대해 연산 횟수를 표현한다.
- **빅 오메가 $$\Omega(n)$$** : 최선일 때의 연산 횟수를 나타낸 표기법
- **빅 세타 $$\theta(n)$$** : 보통일 때의 연산횟수를 나타낸 표기법
- **빅 오 $$O(n)$$** : 최악일 때의 연산횟수를 나타낸 표기법

대부분의 문제는 최악의 경우를 대비해야하므로 **Big O 표기법**을 중심으로 알고리즘 성능을 평가하는 것이 일반적이다.

## 시간 복잡도 유형
### 1. O(1) 상수 시간 복잡도
입력 크기와 관계없이 항상 일정한 시간이 소요된다. 예를 들어 배열의 첫번째 요소에 접근하는 것은 입력 크기와 상관없이 결과를 반환한다.
```java
int getFirstElement(int[] arr) {
    return arr[0];  // O(1)
}

```
### 2. O(n) 선현 시간 복잡도
입력 크기(n)에 비례하여 시간이 증가하는 경우이다. 배열 전체를 순회하여 특정 값을 찾는 알고리즘이 이에 해당한다.
```java
int findElement(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) {
            return i;  // O(n)
        }
    }
    return -1;
}

```
### 3. O(log n) 로그 시간 복잡도
데이터가 절반씩 줄어들면서 탐색하는 경우이다. 대표적으로 **이진탐색** 알고리즘을 예로 들수 있다.
```java
int binarySearch(int[] arr, int target) {
    int left = 0;
    int right = arr.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) {
            return mid;  // O(log n)
        } else if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return -1;  // 값을 찾지 못한 경우
}

```

### 4. O$$(n^2)$$ 이차 시간 복잡도
이중 for문과 같은 중첩된 반복문에서 발생하는 시간 복잡도이다. 입력 크기(n)이 커질 수록 시간이 제곱에 비례해 증가한다. 대표적인 예는 배열 내 모든 쌍을 비교하는 경우이다. 

```java
void printAllPairs(int[] arr) {
    for (int i = 0; i < arr.length; i++) {
        for (int j = i + 1; j < arr.length; j++) {
            System.out.println(arr[i] + ", " + arr[j]);  // O(n²)
        }
    }
}

```
## 다양한 시간 복잡도 비교
입력 크기가 커질 때 시간 복잡도가 성능에 미치는 영향을 쉽게 이해할 수 있도록 아래의 표와 같이 간단한 비교표를 제공할 수 있다.
![](https://velog.velcdn.com/images/duwnstj12/post/a22b9417-6cb4-40b1-aa3f-5cb64f5bca80/image.png)
이 표를 통해 O(1)이나 O(log n)알고리즘은 입력 크기가 커져도 거의 영향을 받지 않지만, O$$(n^2)알고리즘은 입력 크기가 조금만 커져도 성능이 크게 저하된다는 사실을 쉽게 확인 할 수 있다.

## 비효율적인 코드의 개선
비효율적인 코드를 찾아내고, 이를 효율적으로 개선하는것이 알고리즘 성능 최적화의 핵심이다. 특히 이중 또는 삼중 for문과 같은 구조를 사용할 때 더 나은 알고리즘을 적용할 수 있는지 생각해봐야한다.

### 비효율적인 코드 (O$$(n^2)$$)

```java
// 이중 for문을 사용한 비효율적인 코드
for (int i = 0; i < arr.length; i++) {
    for (int j = i + 1; j < arr.length; j++) {
        if (arr[i] + arr[j] == target) {
            return true;
        }
    }
}
return false;

```
위 코드는 배열의 모든 쌍을 비교하므로 O$$(n^2)의 시간이 걸린다 이는 입력 크기가 커질 수록 매우 비효율적이기 때문에 성능이 떨어진다.

### 개선된 코드 O(n)
```java
// HashSet을 사용한 효율적인 코드
Set<Integer> seen = new HashSet<>();
for (int num : arr) {
    if (seen.contains(target - num)) {
        return true;  // 찾으면 바로 종료
    }
    seen.add(num);
}
return false;

```
이 코드에서는 `HashSet`을 사용하여 한번의 배열 순회로 값을 찾는다. 이를 통해 시간 복잡도를 O(n)으로 줄일 수 있고 입력 크기가 커져도 성능이 안정적이다.  

## 요약
1. **알맞은 알고리즘을 선택** : 알고리즘을 선택할 때 시간 복잡도를 고려하는 것이 중요하다. 문제의 입력 크기와 제약조건을 바탕으로 가장 효율적인 알고리즘을 선택해야한다.
2. **비효율적인 코드 개선** : 이중 혹은 삼중 for문 처럼 시간이 오래 걸리는 코드를 찾아 더 효율적인 방법으로 개선하는 것이 중요하다. 자료 구조나 알고리즘을 개선하면 성능을 크게 향상 시킬 수 있다.
