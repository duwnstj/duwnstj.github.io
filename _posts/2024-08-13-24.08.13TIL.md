- 이렇게 명세서를 만들어봤는데 요구사항을 구현하면서도 계속해서 수정을 해줘야할 것 같다. 

## 개인 과제를 하며 겪은 문제점
- 3 계층 아키텍처(Controller,Service,Repository)에 맞게 계층을 설계하고 코드를 짜던 중 create_at(생성 날짜) , update_at(수정 날짜)를 어떻게 설계를 해야할까 하는 난관에 부딪혔다.
- 구글링을 하여 `Timestamp` 타입을 사용하면 된다는 것을 알게 되었다.
- 따라서 아래와 같이 쿼리문을 작성하였다.
```mysql
create table plan (
    id bigint primary key auto_increment, /*일정 고유 번호*/
    todo varchar(400),   /* 할일 */
    name varchar(100),  /* 담당자이름 */
    password varchar(500),/*비밀번호*/
    create_at timestamp default current_timestamp, /* 만든 날짜 */
    update_at timestamp default current_timestamp on update current_timestamp   /* 업데이트 날짜 */
);
```text
## 쿼리문 설명
- 이 쿼리는 plan이라는 이름의 테이블을 생성한다.

- id: bigint 타입의 기본 키(primary key)로, 각 레코드를 유일하게 식별하는 고유 번호이다. auto_increment 옵션을 사용하여 새로운 레코드가 삽입될 때마다 자동으로 증가한다.

- todo: varchar(400) 타입으로, 할일을 저장한다. 최대 400자의 문자열을 저장할 수 있다.

- name: varchar(100) 타입으로, 담당자의 이름을 저장한다. 최대 100자의 문자열을 저장할 수 있다.

- password: varchar(500) 타입으로, 비밀번호를 저장한다. 최대 500자의 문자열을 저장할 수 있다.

- create_at: timestamp 타입으로, 레코드가 생성된 날짜와 시간을 저장한다. default current_timestamp 옵션을 사용하여 레코드가 삽입될 때 자동으로 현재 날짜와 시간으로 설정된다.

- update_at: timestamp 타입으로, 레코드가 마지막으로 수정된 날짜와 시간을 저장한다. `default current_timestamp on update current_timestamp` 옵션을 사용하여 레코드가 삽입될 때는 현재 날짜와 시간으로 설정되고, 레코드가 수정될 때마다 자동으로 현재 날짜와 시간으로 업데이트된다.

### timestamp 타입의 옵션 설명

- default current_timestamp: 새 레코드가 삽입될 때 현재 날짜와 시간으로 자동 설정된다.
- on update current_timestamp: 레코드가 수정될 때마다 현재 날짜와 시간으로 자동 갱신된다.

이 설정을 통해 create_at 열은 레코드가 처음 생성될 때의 시간을 저장하고, update_at 열은 레코드가 마지막으로 수정된 시간을 기록하게 된다.
