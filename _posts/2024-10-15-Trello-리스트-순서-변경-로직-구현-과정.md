- 처음 순서 변경 로직을 구현할 때 **리스트 항목의 순서를 어떻게 효율적으로 변경할지**에 대한 고민이 있었습니다. 자바 학습중에 배운**스왑 방식**을 떠올려 임시 변수를 활용하여 두 항목의 순서를 교체하는 방법을 사용해 코드를 작성했었습니다.
## 기존 서비스 코드
```java
 @Transactional
    public List<ListUpdateResponse> updateOrderNumbers(AuthUser authUser, ListUpdateRequest request) {

        Lists changelist = listRepository.findByOrderNumber(request.getOrderNumber())
                .orElseThrow(ListNotFoundException::new);
        Lists secondchange = listRepository.findByOrderNumber(request.getNewOrderNumber())
                        .orElseThrow(ListNotFoundException::new);
        Long templistorderNumber = changelist.getOrderNumber();

        changelist.update(secondchange.getOrderNumber());
        secondchange.update(templistorderNumber);

        List<Lists> changedList = listRepository.orderNumberAsc();

        return changedList.stream().map(list -> new ListUpdateResponse(
                        list.getTitle(),
                        list.getOrderNumber()
                ))
                .collect(Collectors.toList());
```
### 코드 설명
- **임시 변수 활용** : `temp` 변수를 사용하여 두 리스트 항목의 순서를 교체하는 방식으로 구현했습니다. 리스트에서 `orderNumber`필드를 사용해 현재 항목의 순서를 저장하고 교체할 항목의 순서와 스왑하는 방식으로 작성했습니다.
- **초기 접근 방식** : 자바에서 흔히 사용하는 스왑 알고리즘을 기반으로 간단하게 구현했지만 실사용 환경(Trello와 같은 서비스)에서는 이 방식이 적절하지 않다는것을 확인했습니다.


## 기존 코드의 문제점

### 기존 코드의 순서 변경 로직 
기존 코드는 두 항목을 교체하는 방식으로 동작합니다.
1. 1번 항목과 4번 항목을 교체합니다.
2. 두 값이 서로 교체되면서 해당 순서가 업데이트됩니다.
![](https://velog.velcdn.com/images/duwnstj12/post/e8f18c0b-384c-4ffa-93ed-e2239bc94705/image.png)

- Trello에서 이 방식을 사용했는지 확인해보니 실제로는 이러한 방식이 아니었습니다. **Trello**에서 항목의 순서를 변경할 때는 단순히 두 항목을 교체하는 것이 아니라 **나머지 항목들도 자연스럽게 밀리는 방식**으로 처리되었습니다.

***

### Trello의 순서 변경 로직
Trello에서는 순서가 다음과 같이 변경됩니다.
- 예를 들어 4번 항목을 1번 위치로 옮기면 순서는 4,1,2,3으로 변경됩니다. 기존 항목들이 밀리면서 전체 순서가 재정렬되는 것입니다.
![](https://velog.velcdn.com/images/duwnstj12/post/4f9c216f-e548-4e3d-bdb5-eb00b2636f12/image.png)

- 이 방식에 맞춰서 새로운 로직을 설계해야했습니다. 따라서 **리스트의 전체 순서를 재정렬하는 방식**을 사용했습니다.


## 두번째로 구현한 서비스 코드
```java
@Transactional
    public List<ListUpdateResponse> updateOrderNumbers(AuthUser authUser, ListUpdateRequest request, Long boardId, Long workspaceId) {

        WorkspaceUser wu = workspaceUserRepository.findByWorkspaceIdAndUserId(workspaceId,authUser.getId())
                .orElseThrow(()-> new WorkspaceUserNotFoundException("해당 워크스페이스에 있는 유저가 아닙니다"));

        // 역할이 readOnly인 경우 Exception 처리
        if(wu.getRole().equals(WorkspaceRole.READ_ONLY)){
            throw new WorkspaceRoleForbiddenException("리스트를 수정할 권한이 없습니다.");
        }

        List<Lists> listsList = listRepository.findAllByBoardId(boardId);

        Lists listToMove = listsList.stream()
                .filter(list -> list.getId().equals(request.getListId()))
                .findFirst()
                .orElseThrow(ListNotFoundException::new);

        listsList.remove(listToMove);

        int newPosition = Math.min(request.getNewPosition(), listsList.size());

        listsList.add(newPosition, listToMove);

        for (int i = 0; i < listsList.size(); i++) {
            Lists list = listsList.get(i);
            Long newOrderNumber = (long) (i + 1);
            if (!list.getOrderNumber().equals(newOrderNumber)) {
                list.update(newOrderNumber);
            }
        }

        return listsList.stream()
                .map(list -> new ListUpdateResponse(
                        list.getTitle(),
                        list.getId(),
                        list.getOrderNumber()
                )).collect(Collectors.toList());

    }
```
### 코드 설명

#### 1. 사용자 권한 확인
- 사용자가 **워크 스페이스에 속한 유저인지 확인**하고 속하지 않으면 `workspaceUserNotFoundException` 예외를 발생시킵니다. 이는 **보안적인 이유**로 필수적인 부분입니다.
- **읽기 전용 권한**인 경우에는 리스트 수정 권한이 없으므로 예외를 처리해주었습니다.

#### 2. 리스트 이동 로직
- 해당 보드의 모든 리스트를 가져온 후 요청으로 전달받은 `listId`에 해당하는 리스트를 찾아내 **기존 리스트에서 제거 후 새로운 위치에 추가합니다**
- 새로운 위치가 리스트의 크기를 초과하지 않도록 `Math.min()`을 사용하여 **인덱스 초과 오류를 방지**합니다.

#### 3. 순서 재정렬
- 변경된 리스트를 이용해 전체 리스트의 순서를 **다시 정렬**합니다.
- 각 리스트의 `OrderNumber`가 현재 인덱스에 맞게 **재정렬되며 순서가 맞지 않는 리스트에 대해서만 업데이트**하여 성능을 최적화합니다.

#### 4. 결과 반환
- 최종적으로 **변경된 리스트들의 정보**를 `ListUpdateResponse` 객체에 담아 반환합니다.

### Trello 방식으로 개선된 순서 변경 로직
새로운 코드에서는 **리스트의 전체적인 순서가 재정렬**되도록 구현했습니다. 아래 그림에서 볼 수 있듯이 **4번 항목을 1번 위치로 이동**시키면 나머지 항목들도 밀려나면서 순서가 업데이트됩니다.
