## 첫번째 문제 풀이 코드(실패)
{% highlight java %}
class Solution {
    public int solution(int left, int right) {
        int answer = 0;
        int count = 0;
        for (int i = left; i <= right; i++) { //1
            
            for (int j = 1; j <= i; j++) { //2
                if (i % j == 0) {
                    count++;
                }
            }
            if (count % 2 == 0) { //3
                answer += i;
            } else { 
                answer -= i;
            }
        }
        return answer;
    }
}

{% endhighlight %}
## 코드 설명
- **주석으로 풀이**
1. `이중 for문`을 이용하여 만약 13일때 13의 약수를 구할 수 있게 코드를 짰다. 바깥 for문에서는 left~right까지 반복을 해주었다.
2. 안쪽 for문에서는 left의 약수를 구해 반복횟수를 count 변수를 이용해 담아주었다.
3. 바깥 for문 안에서 만약 count가 짝수일 때 answer에 i(left 값)을 더해주고, 홀수라면 answer에 i(left 값)을 빼주어 문제를 풀었다.
- 하지만 로직은 맞는것같은데 내가 원하는 값이 아닌 다른 값이 나오는 걸 보고 다시 코드를 분석해봤다.

## 코드의 문제점
- 원하는 값이 안나오던 이유는 바로 count가 약수를 count하고 바깥 for문으로 나올 때 다시 0으로 초기화가 되어야 다음 left의 약수를 정확하게 count 할 수 있는데 count 변수를 for문 밖에서 선언을 해주니 값이 이상하게 나오는것이었다.
## 두번째 문제 풀이 코드(성공)
{% highlight java %}
class Solution {
    public int solution(int left, int right) {
        int answer = 0;
        for(int i=left;i<=right;i++){
            int count=0;
            for(int j=1;j<=i;j++){
                if(i % j ==0){
                    count++;
                }
            }
            if(count % 2 ==0){
                answer+= i;
            }else{
                answer-= i;
            }
        }
        return answer;
        }
        
    }

{% endhighlight %}
- 따라서 두번째 코드는 count 변수를 바깥 for문 안에서 0으로 초기화 해주었다.
- 그 후 문제 제출을 해보니 성공 하였다.
