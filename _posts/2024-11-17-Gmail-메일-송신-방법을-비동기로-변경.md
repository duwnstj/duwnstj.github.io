## 개요

`AOP`를 활용해 책 대여 알림을 Gmail로 송신하는 기능을 구현했습니다. 하지만 **포스트맨(Postman)**을 통해 테스트한 결과, 응답 속도가 느린 문제가 발생했습니다.

아래 이미지를 보면 약 5초 정도가 소요되는 것을 확인할 수 있습니다.
![](https://velog.velcdn.com/images/duwnstj12/post/5122b06c-cad0-407b-8183-c1b0e18b3e7a/image.png)

이 문제는 **동기 방식**으로 이메일을 전송하면서 메인 쓰레드가 작업이 끝날때까지 블로킹(blocking)이 되기 때문에 발생했습니다.

이를 해결하기 위해 **비동기 방식**을 사용하여 메일 송신 방식을 개선했습니다.

## 기존 코드

{% highlight java %}
 //실제 이메일 전송 로직
    public void sendEmail(String toEmail, String message) {

        SimpleMailMessage mailMessage = new SimpleMailMessage();
        mailMessage.setTo(toEmail);
        mailMessage.setSubject("도서관 알림");
        mailMessage.setText(message);

        mailSender.send(mailMessage);

        log.info("이메일을 성공적으로 보냈습니다.");
    }

{% endhighlight %}
- 위 코드는 **동기 방식**으로 작성되어 이메일 송신 작업이 완료될 때까지 **메인 쓰레드가 대기**합니다.
- 이로 인해, 메일 송신 시간이 API 응답 속도에 직접적인 영향을 미쳤습니다.

## 비동기 방식 도입

## AsyncConfig 작성
비동기 작업을 활성화하려면 `@EnableAsync`를 적용하고 스레드 풀을 설정해야합니다.
{% highlight java %}
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.scheduling.annotation.EnableAsync;
import org.springframework.scheduling.concurrent.ThreadPoolTaskExecutor;

import java.util.concurrent.Executor;
import java.util.concurrent.ThreadPoolExecutor;

@Configuration
@EnableAsync
public class AsyncConfig {
    @Bean
    public Executor asyncExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(5); // 최소 스레드 수
        executor.setMaxPoolSize(10); // 최대 스레드 수
        executor.setQueueCapacity(100); // 대기열 크기
        executor.setThreadNamePrefix("AsyncNotification-");
        executor.initialize();
        return executor;
    }
}
{% endhighlight %}
- `@EnableAsync` : 비동기 작업을 활성화합니다.
## 비동기 로직 적용
`@Async`를 사용해 메일 송신 로직을 비동기로 처리하도록 변경했습니다.

{% highlight java %}
 //실제 이메일 전송 로직
    @Async
    public void sendEmail(String toEmail, String message) {
    try {
        SimpleMailMessage mailMessage = new SimpleMailMessage();
        mailMessage.setTo(toEmail);
        mailMessage.setSubject("도서관 알림");
        mailMessage.setText(message);

        mailSender.send(mailMessage);

        log.info("이메일을 성공적으로 보냈습니다.");
    }catch (MailException e){
        log.error("이메일 전송 에러 발생: {}", e.getMessage(),e);
    }

    }
{% endhighlight %}
**변경 사항** 
1. `@Async` 사용
    - 메일 송신 작업을 비동기로 실행해 **메인 쓰레드가 블로킹되지 않도록 처리했습니다.**
    - 이메일 송신이 완료되기 전에도 API 응답이 즉시 반환됩니다.
2. `try-catch` 블록 추가
    - 이메일 송신 중 발생할 수 있는 예외를 처리하고 에러 로그를 기록합니다.
    - 에러 로그를 통해 유지보수와 디버깅이 쉬워집니다.

## 결과
비동기 방식을 적용한 후 포스트맨으로 다시 테스트를 진행했습니다.

- 개선 후 : **약 440ms**
