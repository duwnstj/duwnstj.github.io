## 컨트롤러 코드
```java
 @GetMapping("/todos")
    public ResponseEntity<Page<TodoResponse>> getTodos(
            @RequestParam(defaultValue = "1") int page,
            @RequestParam(defaultValue = "10") int size,
            @RequestParam(required = false) String weather,
            @RequestParam(required = false)LocalDateTime startDate,
            @RequestParam(required = false) LocalDateTime endDate
            ) {
        return ResponseEntity.ok(todoService.getTodos(page, size,weather,startDate,endDate));
    }
```text
### 첫 번째 요구사항
- 첫 번째 요구사항을 해결하기 위해 ` @RequestParam(required = false) String weather`를 통해 `weather` 값을 쿼리 파라미터로 받도록 설정했다. `@RequestParam`은 HTTP 요청의 **쿼리 파라미터**를 매핑하는데 사용되며,쿼리 파라미터는 아래와 같은 형식으로 전달 할 수 있다. 

 ```java
 http://localhost:8080/todos?weather=Snowy and Cold
 ``` 
 
- **쿼리파라미터**는 키-값 쌍으로 이루어져 있다. 위의 예에서는 `weather`가 키이고 , `Snowy and Cold`가 그에 대한 값이다.
- PostMan을 사용하여 쿼리 파라미터를 쉽게 설정할 수 있다.

![](https://velog.velcdn.com/images/duwnstj12/post/b9982754-0413-46b3-8509-2b18c52a6784/image.png)
- 여기서 `required` 속성을 false로 설정한 이유는 `@RequestParam`이 기본적으로 필수 파라미터로 설정되어 있기 때문이다. 만약 `required = false`를 설정하지 않으면 해당 파라미터가 요청에서 누락될 경우, 오류가 발생한다. 
    - 따라서 이 속성을 설정하여 파라미터가 없더라도 요청이 정상적으로 처리되도록 하였다.
   
### 두 번째 요구사항
- 두 번째 요구사항을 해결하기 위해 `@RequestParam`으로 `startDate`와 `endDate`를 받아 일정 기간을 필터링할 수 있도록 구현했다. 
- 이 값들은 `LocalDateTime` 클래스로 받고 있으므로 요청을 보낼 때는 ISO 8601 형식 `(yyyy-MM-dd'T'HH:mm:ss)`으로 보내야 한다. 예를 들면 다음과 같다.

![](https://velog.velcdn.com/images/duwnstj12/post/cba4411b-f082-43d2-855b-030a5d358a16/image.png)

- 이렇게 날짜와 시간을 모두 보내는 이유는 **`LocalDateTime`이 연도,월,일,시간까지 모두 필요로하기 때문이다.** 
- 이를 통해 더 정밀한 날짜와 시간 범위를 설정할 수 있다.

## 서비스 코드
```java
  public Page<TodoResponse> getTodos(int page, int size, String weather, LocalDateTime startDate, LocalDateTime endDate) {
        Pageable pageable = PageRequest.of(page - 1, size);
        Page<Todo> todos;
        if (weather == null || weather.isEmpty()) {
            todos = todoRepository.findAllByOrderByModifiedAtDesc(pageable);
        }else {
            todos = todoRepository.findAllByWeather(weather, pageable);
        }

        if (startDate !=null || endDate != null) {
            todos = todoRepository.findSchedulesBetweenDates(pageable,startDate,endDate);
        }
```text
### 설명
- 첫번째 조건에서 `weather` 값을 필터링한다. `weather`가 없으면 모든 일정 데이터를 조회하고, `weather`가 있을 경우 해당 조건으로 필터링한다.
- `startDate`와 `endDate`가 존재하면 해당 기간 내의 일정을 조회한다.
- 현재 요구사항에서는 두가지 경우(날씨 조건 또는 기간 조건)만을 처리하고, 복합 조건을 처리할 필요가 없어서 각 조건을 독립적으로 처리하는 방식으로 구현했다. 
- 추후 복합적인 필터 조건이 필요하게 되면 추가적인 로직을 더해 확장 할 수 있다.

## Repository 코드
```java
package org.example.expert.domain.todo.repository;

import org.example.expert.domain.todo.entity.Todo;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.Pageable;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;

import java.time.LocalDateTime;
import java.util.Optional;

public interface TodoRepository extends JpaRepository<Todo, Long> {

    @Query("SELECT t FROM Todo t LEFT JOIN FETCH t.user u ORDER BY t.modifiedAt DESC")
    Page<Todo> findAllByOrderByModifiedAtDesc(Pageable pageable);

    @Query("SELECT t FROM Todo t " +
            "LEFT JOIN t.user " +
            "WHERE t.id = :todoId")
    Optional<Todo> findByIdWithUser(@Param("todoId") Long todoId);

    //weather를 기준으로 일정 전체 검색
    @Query("select t from Todo t where t.weather = :weather")
    Page<Todo> findAllByWeather(@Param("weather") String weather, Pageable pageable);

    //수정일을 기준으로 특정 기간 일정 찾기
    @Query("select t from Todo  t where t.modifiedAt >= :startDate And t.modifiedAt<= :endDate order by t.modifiedAt desc")
    Page<Todo> findSchedulesBetweenDates(Pageable pageable,
                                         @Param("startDate") LocalDateTime startDate,
                                         @Param("endDate") LocalDateTime endDate);

}
```text
### 첫 번째 요구사항
- `weather`를 기준으로 일정을 검색하기 위해 JPQL을 사용했다.
```java
  //weather를 기준으로 일정 전체 검색
    @Query("select t from Todo t where t.weather = :weather")
    Page<Todo> findAllByWeather(@Param("weather") String weather, Pageable pageable);
```text
- 이 쿼리문은 @Param을 사용해 쿼리 파라미터로 전달받은 `weather` 값을 바인딩하여 해당 조건을 만족하는 일정을 조회한다.
- `:weather`는 JPQL에서 파라미터를 사용하는 방식이다.

### 두 번째 요구사항
- 일정의 수정일을 기준으로 특정 기간의 일정을 조회하기 위해 JPQL을 사용했다.

```java
//수정일을 기준으로 특정 기간 일정 찾기
    @Query("select t from Todo  t where t.modifiedAt >= :startDate And t.modifiedAt<= :endDate order by t.modifiedAt desc")
    Page<Todo> findSchedulesBetweenDates(Pageable pageable,
                                         @Param("startDate") LocalDateTime startDate,
                                         @Param("endDate") LocalDateTime endDate);
```text
- 이 쿼리문은 `startDate`와 `endDate` 사이에 있는 일정을 조회한다. `modifiedAt` 필드를 기준으로 `startDate` 이후 `endDate` 이전의 데이터를 조회한다.(특정 기간을 조회)
`orderby t.modifiedAt desc`를 통해 최신 수정일 순으로 정렬한다.
