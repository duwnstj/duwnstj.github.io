**QueryDSL**은 JPA와 함께 사용하는 도구로 자바 코드로 안전하게 쿼리를 작성할 수 있게 해줍니다. 특히 복잡한 조건을 처리하는 **동적 쿼리**를 작성할 때 유용합니다 이 글에서는 **queryFactory** 설정부터 동적 쿼리 작성까지 과정을 상세히 설명하겠습니다.

### 1. QueryDSL의 기본 개념
QueryDSL은 자바 코드로 JPA 쿼리를 작성할 수 있는 프레임워크입니다. QueryDSL은 각 엔티티에 대한 **Q클래스**를 자동으로 생성하여 이를 통해 쿼리를 작성합니다.

{% highlight java %}
QMember member = QMember.member;

{% endhighlight %}
위 코드에서 `QMember`는 `Member` 엔티티의 필드에 접근할 수 있는 클래스입니다. 이 클래스를 통해 필드에 접근하여 쿼리를 작성할 수 있습니다. 

### 2. JPAQueryFactory 설정하기

QueryDSL에서 동적 쿼리를 작성하려면 **JPAQueryFactory**를 빈으로 등록하고, 리포지토리나 서비스 클래스에서 의존성 주입을 통해 사용해야합니다. `JPAQueryFactory`는 쿼리 실행에 필요한 **EntityManager**를 사용합니다.

#### (1) JPAQueryFactory 빈 등록

먼저 `JPAQueryFactory`를 스프링 빈으로 등록하는 설정이 필요합니다. 일반적으로 `@Configuration` 클래스에서 등록합니다.

{% highlight java %}
@Configuration
public class QuerydslConfig {

    @PersistenceContext  // JPA의 EntityManager 주입
    private EntityManager entityManager;

    @Bean  // JPAQueryFactory를 빈으로 등록
    public JPAQueryFactory queryFactory() {
        return new JPAQueryFactory(entityManager);
    }
}

{% endhighlight %}
- **EntityManager**는 JPA에서 데이터베이스와 상호작용하는 객체로, **JPAQueryFactory**가 이를 사용해 쿼리를 실행 할 수 있습니다.

#### (2) JPAQueryFactory 주입 받기

이제 `JPAQueryFactory`를 서비스나 리포지토리에서 주입받아 동적쿼리를 작성할 수 있습니다. 

{% highlight java %}
@Service
public class MemberService {

    private final JPAQueryFactory queryFactory;

    // 생성자를 통해 JPAQueryFactory 주입
    @Autowired
    public MemberService(JPAQueryFactory queryFactory) {
        this.queryFactory = queryFactory;
    }

    public List<Member> searchMembers(String name, Integer age) {
        QMember member = QMember.member;
        BooleanBuilder builder = new BooleanBuilder();  // 동적 조건 작성

        if (name != null) {
            builder.and(member.name.eq(name));  // 이름 조건 추가
        }
        if (age != null) {
            builder.and(member.age.eq(age));  // 나이 조건 추가
        }

        return queryFactory
            .selectFrom(member)
            .where(builder)  // 조건 적용
            .fetch();  // 결과 반환
    }
}

{% endhighlight %}
- **JPAQueryFactory 주입** : 생성자를 통해 `JPAQueryFactory`를 주입받아 동적쿼리를 작성합니다.
- **BooleanBuilder** : 조건이 있을 때만 동적으로 쿼리에 추가하도록하는 객체입니다.

### 3. 동적 쿼리 작성
`queryFactory`를 사용해 다양한 동적 쿼리를 작성할 수 있습니다. **BooleanBuilder**를 사용해 여러 조건을 추가하고 이를 통해 동적으로 쿼리를 변경 할 수 있습니다.

{% highlight java %}
public List<Member> searchMembers(String name, Integer age) {
    QMember member = QMember.member;
    BooleanBuilder builder = new BooleanBuilder();

    if (name != null) {
        builder.and(member.name.eq(name));  // 이름 조건
    }
    if (age != null) {
        builder.and(member.age.eq(age));  // 나이 조건
    }

    return queryFactory
        .selectFrom(member)
        .where(builder)  // 조건 적용
        .fetch();  // 결과 반환
}

{% endhighlight %}
### 4. 조인,페이징,정렬 적용
QueryDSL은 조인,페이징,정렬 같은 복잡한 기능도 간단하게 처리할 수 있습니다.

#### (1) 조인 처리

다른 테이블과의 조인도 쉽게 처리할 수 있습니다. 

{% highlight java %}
public List<Order> findOrdersWithMembers(String memberName) {
    QOrder order = QOrder.order;
    QMember member = QMember.member;

    return queryFactory
        .selectFrom(order)
        .join(order.member, member)  // order와 member 테이블 조인
        .where(member.name.eq(memberName))  // 회원 이름 조건
        .fetch();
}

{% endhighlight %}
#### (2) 페이징과 정렬 처리

페이징과 정렬도 쉽게 처리할 수 있습니다. 

{% highlight java %}
public List<Member> searchMembersWithPaging(int page, int size) {
    QMember member = QMember.member;

    return queryFactory
        .selectFrom(member)
        .orderBy(member.name.asc())  // 이름 오름차순 정렬
        .offset(page * size)  // 페이지 시작점 설정
        .limit(size)  // 한 페이지에 표시할 데이터 수 설정
        .fetch();
}

{% endhighlight %}
## 꼬리 지문 예상

### 1. Queery DSL을 사용하면 성능에 문제가 없나요?
- **QueryDSL 자체는 성능 문제를 유발하지 않지만** 잘못된 쿼리 작성이나 과도한 조인을 사용하면 성능 저하가 발생할 수 있습니다. 특히 `Fetch Join`을 사용할 떄 주의해야하며 ,필요하지 않은 데이터를 너무 많이 가져오지 않도록 쿼리를 최적화하는 것이 중요합니다. 

### 2. QueryDSL로 N+1 문제를 해결 할 수 있나요?

- 해결 할수 있습니다. **Fetch Join**을 사용해 연관된 엔티티들을 한번에 가져옴으로써 **N+1 문제**를 해결할 수 있습니다. 하지만 Fetch Join은 필요한 데이터만 가져오도록 설정하지 않으면 성능에 영향을 미칠 수 있으므로 주의가 필요합니다.

### 3. QueryDSL로 복잡한 동적 쿼리를 작성할 떄 주의해야할 점은 무엇인가요?
- **쿼리의 복잡성과 성능 최적화**를 고려해야합니다. 잘못된 쿼리 작성이나 지나치게 많은 조인 사용은 성능 문제를 일으킬 수 있으므로 쿼리 실행 계획을 확인하고 성능을 최적화하는것이 중요합니다.
