## 문제 
- QueryDSL을 통해 동적쿼리를 작성한 후 실행했는데 이때 `NullPointerException`이 발생했습니다.

## 에러 코드
```java
2024-10-23T11:31:10.067+09:00 ERROR 24556 --- [library_management] [nio-8080-exec-1] o.a.c.c.C.[.[.[/].[dispatcherServlet]    : Servlet.service() for servlet [dispatcherServlet] in context with path [] threw exception [Request processing failed: java.lang.NullPointerException: Cannot invoke "java.lang.Integer.intValue()" because "reviewStar" is null] with root cause

java.lang.NullPointerException: Cannot invoke "java.lang.Integer.intValue()" because "reviewStar" is null

```text
## 첫 번째 트러블 슈팅
- 이 에러가 컨트롤러에서 발생하는지 서비스에서 발생하는지 확인하기 위해 log.info()를 사용하여 로그를 찍어봤습니다. 로그가 서비스 쪽에서 찍히지 않은 것을 확인하고, 컨트롤러에서 `NullPointerException`이 발생했다는 것을 알 수 있었습니다.


## 문제가 발생한 컨트롤러 코드
```java
  /**
     * reviewStar와 bookId가 존재할때 해당 책에 대한 별점을 기준으로 조회
     * 별점과 책 고유 번호가 null 값일때 리뷰 전체 조회
     * 별점만 null 일때 해당 책의 리뷰 전체 조회
     * bookId만 null일때 별점이 1~5점인 책을 조회
     *
     * @param page       : 현재 페이지
     * @param size       : 페이지 크기
     * @param bookId     : 책 고유 번호
     * @param reviewStar : 별점
     * @return response, Httpstatus(200) : 리뷰 정보 , 성공 상태메시지
     */
    @GetMapping("/reviews")
    public ResponseEntity<Page<ReviewsGetResponse>> findAllByMultipleConditions(
            @RequestParam(defaultValue = "1") int page,
            @RequestParam(defaultValue = "10") int size,
            @RequestParam(required = false) Long bookId,
            @RequestParam(required = false) int reviewStar
    ) {
        log.info("잘 들어오는지 확인 중 ,,,");
        return ResponseEntity.ok(reviewService.findAllByMultipleConditions(page, size, bookId, reviewStar));

    }
```text
- 위의 코드를 보면 `@RequestParam`으로 `reviewStar(별점)`을 받아오는데 `required=false`로 설정되어있습니다. 이 속성은 Default 값이 true인데, false로 설정하면 이 값이 없어도 `null` 값으로 처리를 해 받아주기때문에 동적 쿼리를 작성하기위해 false로 설정을 해두었습니다.
- 하지만 문제는 `int`가 원시형 타입이라 `null 값`을 허용하지 않는다는 점입니다. 따라서 `int` 타입 대신, `Integer`로 변경해줘야 null 값 처리를 안전하게 
할 수 있습니다.

- 이렇게 컨트롤러 쪽 문제를 해결했지만, 다시 `NullPointerException`이 발생했습니다. 이번에는 서비스 로직에서 발생한 것이었습니다.


## 두번째 트러블 슈팅
- 이번에는 서비스 쪽에 작성한 별점 유효범위 검증하는 코드에서 발생했습니다.

### 문제가 발생한 서비스 코드
```text
// 조건에 맞는 리뷰 리스트 조회
    public Page<ReviewsGetResponse> findAllByMultipleConditions(int page,
                                                                int size,
                                                                Long bookId,
                                                                Integer reviewStar) {
        Pageable pageable = PageRequest.of(page - 1, size);


        // 별점이 null 이 아닐 경우에만 유효 범위 검증
        if (!(reviewStar >= 1 && reviewStar <= 5)) {
            throw new InvalidReviewStarException();
        }

        // 별점이 null이면 그책에 해당하는 전체 별점을 조회하고 값이 있으면 해당 별점만 조회
        return reviewRepository.findAllByMultipleConditions(
                pageable, bookId, reviewStar);


    }
```text
### 문제가 발생한 이유
- 위 코드를 보면 `reviewStar`가 `null`일 때, `1~5` 범위의 숫자와 비교하려고 해서 `NullPointerException`이 발생한 것입니다. 

- `Integer`는 `int`와 비교할 때 자동으로 언박싱되어 원시형 `int`로 변환되지만, `reviewStar`가 `null`이면 이를 언박싱할 수 없기 때문에 예외가 발생한 것입니다.


### 수정한 코드
```java
 // 별점이 null 이 아닐 경우에만 유효 범위 검증
        if (reviewStar != null && !(reviewStar >= 1 && reviewStar <= 5)) {
            throw new InvalidReviewStarException();
        }
```text
이렇게 `reviewStar`가 `null`인지 먼저 확인한 후 유효 범위 검증을 하도록 수정하여 문제를 해결했습니다.
