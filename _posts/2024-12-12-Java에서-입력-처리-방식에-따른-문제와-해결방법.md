Java를 사용해 문제를 해결할 때, 주어진 입력 형식에 따라 데이터를 읽고 처리하는 방식이 중요합니다. 이번 포스트에서는 한 문제를 통해 입력 형식과 처리 방식의 불일치로 발생한 문제를 분석하고, 이를 어떻게 해결했는지 알아보겠습니다.

---

## 🎶 문제
### 문제 설명
총 `N`개의 정수가 주어졌을 때, 특정 정수 `v`가 몇 번 등장하는지 구하는 프로그램을 작성하시오.

### 입력 형식
1. 첫 번째 줄: 정수의 개수 `N` (1 ≤ N ≤ 100)
2. 두 번째 줄: `N`개의 정수가 공백으로 구분되어 주어짐
3. 세 번째 줄: 찾으려고 하는 정수 `v`

### 출력 형식
`N`개의 정수 중에서 `v`가 몇 번 등장하는지를 출력한다.

## **첫 번째 시도 코드**
다음은 문제를 해결하기 위해 작성한 첫 번째 코드입니다.

```java
import java.io.*;
import java.util.StringTokenizer;

public class Main {
    public static void main(String[] args) throws IOException {
        try (BufferedReader bufferedReader = new BufferedReader(new InputStreamReader(System.in));
             BufferedWriter bufferedWriter = new BufferedWriter(new OutputStreamWriter(System.out))) {

            int N = Integer.parseInt(bufferedReader.readLine());
            int[] numbers = new int[N];

            // 정수를 배열에 채우기
            for (int i = 0; i < numbers.length; i++) {
                StringTokenizer stringTokenizer = new StringTokenizer(bufferedReader.readLine());
                numbers[i] = Integer.parseInt(stringTokenizer.nextToken());
            }

            int target = Integer.parseInt(bufferedReader.readLine());
            int count = 0;

            for (int number : numbers) {
                if (number == target) {
                    count++;
                }
            }

            bufferedWriter.write(count + "\n");
            bufferedWriter.flush();
        }
    }
}
```

### **이 코드를 사용한 이유**
이 코드는 BufferedReader로 입력받은 값들을 `readLine()` 메서드를 통해 한 줄씩 읽고, `StringTokenizer`를 사용해 공백 또는 줄바꿈으로 나눠 값을 배열에 저장합니다.

## **문제점**
![입력 형식 오류](https://velog.velcdn.com/images/duwnstj12/post/19eb21bb-7c3c-40fe-96a6-33cb1e693b05/image.png)

위 코드는 실행 중 **`NoSuchElementException`** 예외가 발생합니다. 그 이유는 입력 방식의 문제 때문입니다.

### **문제의 원인**
1. `bufferedReader.readLine()`은 한 번 호출할 때마다 한 줄을 읽어옵니다.
2. 반복문 내부에서 `readLine()`을 호출하면 **다음 줄**로 이동하며 데이터를 읽습니다.
3. 하지만 문제의 둘째 줄은 공백으로 구분된 정수들이 **한 줄**에 입력됩니다.
4. 위 코드에서는 매 반복마다 새로운 줄을 읽으려 하기 때문에 데이터가 부족하게 되어 예외가 발생합니다.

### **입력 형식과의 불일치**
문제에서 제공되는 입력은 다음과 같습니다:
```
5
1 2 3 2 2
2
```

하지만 위 코드의 동작 방식은 다음과 같은 입력을 처리하도록 작성되었습니다:
```
5
1
2
3
2
2
2
```

따라서 입력 형식과 코드가 맞지 않아 올바른 결과를 얻지 못했습니다.

## **두 번째 코드 (수정된 코드)**
문제의 입력 형식에 맞춰 둘째 줄에서 공백으로 구분된 모든 정수를 한 번에 읽어 처리하도록 코드를 수정했습니다.

```java
import java.io.*;
import java.util.StringTokenizer;

public class Main {
    public static void main(String[] args) throws IOException {
        try (BufferedReader bufferedReader = new BufferedReader(new InputStreamReader(System.in));
             BufferedWriter bufferedWriter = new BufferedWriter(new OutputStreamWriter(System.out))) {

            int N = Integer.parseInt(bufferedReader.readLine());
            int[] numbers = new int[N];

            // 둘째 줄의 모든 정수를 한 줄로 읽어 처리
            StringTokenizer stringTokenizer = new StringTokenizer(bufferedReader.readLine());
            for (int i = 0; i < N; i++) {
                numbers[i] = Integer.parseInt(stringTokenizer.nextToken());
            }

            int target = Integer.parseInt(bufferedReader.readLine());
            int count = 0;

            for (int number : numbers) {
                if (number == target) {
                    count++;
                }
            }

            bufferedWriter.write(count + "\n");
            bufferedWriter.flush();
        }
    }
}
```

### **이 코드를 사용한 이유**
문제의 입력 형식에 맞게, 둘째 줄에서 **한 번에 모든 정수를 읽고** 공백을 기준으로 나눠 배열에 저장했습니다. 이 방식은 **공백으로 구분된 정수들이 한 줄로 주어진 경우에 적합**합니다.

## **첫 번째 코드와 두 번째 코드의 차이점**

### **첫 번째 코드의 실패 원인**
1. 첫 번째 코드는 **각 정수를 새로운 줄에서 읽는 형식**을 가정했습니다.
2. 하지만 실제 문제는 정수들이 **공백으로 구분된 한 줄**로 제공됩니다.
3. `readLine()`을 반복 호출하면서 데이터 부족 문제가 발생했습니다.

### **두 번째 코드의 성공 이유**
1. 입력 형식에 맞춰 **한 줄로 주어진 모든 정수를 한꺼번에 읽고** 처리하도록 수정했습니다.
2. 이를 통해 데이터 부족 문제를 해결하고 올바른 결과를 출력할 수 있었습니다.
