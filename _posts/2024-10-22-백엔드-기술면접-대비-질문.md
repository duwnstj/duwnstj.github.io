# Java 관련 질문

## 자바의 주요 특징은 무엇인가요?
자바는 **객체 지향 언어**로 캡슐화,상속,다형성,추상화의 OOP원칙을 따르며 유지보수성과 재사용성이 뛰어납니다. 또한 **플랫폼 독립성**을 제공하여 JVM에서 실행되므로 운영체제에 상관없이 실행할 수 있습니다. 자바는 **Garbage Collection**을 통해 자동 메모리 관리를 제공하고 **강타입 언어**로 컴파일시 타입 검사를 강화하여 안정성을 높입니다.

### 꼬리질문 1: 자바에서의 플랫폼 독립성이 어떻게 실현되나요?
자바는 소스코드를 바이트 코드로 컴파일하여 운영체제와 무관하게 **JVM**에서 실행됩니다. JVM이 바이트코드를 각 운영체제에 맞게 바이트 코드를 해석하여 실행하므로 같은 바이트코드가 다른 운영체제에서 동작할 수 있습니다.

### 꼬리 질문 2 : 자바의 Garbage Collection은 어떻게 동작하나요?
Garbage Collection은 참조되지 않는 객체를 자동으로 탐지하여 메모리를 회수하는 기능입니다. JVM은 사용되지 않는 객체를 자동으로 정리하여 메모리 누수 없이 효율적으로 메모리를 관리합니다.

### 꼬리 질문 3 : 자바에서 객체 지향 프로그래밍(OOP)란 무엇인가요?
객체 지향 프로그래밍(OOP)은 객체라는 단위로 프로그램을 구성하는 방식으로 자바는 **캡슐화,상속,다형성,추상화** 원칙을 따릅니다. 이를 통해 코드의 **재사용성,유지 보수성**을 높일 수 있습니다.

### 꼬리질문 4: 객체 지향의 4대원칙중 하나인 다형성에 대해 설명해주세요
다형성은 객체 지향 프로그래밍의 핵심 개념 중 하나로 **같은 메서드나 연산자가 다른 객체**에서 다르게 동작할 수 있는 능력을 의미합니다. 자바에서는 **메서드 오버로딩(메서드 매개변수를 다르게 하여 같은 메서드 이름을 여러개 사용)**과 **오버라이딩(메서드를 상속받아 재정의해 사용)**으로 다형성을 구현할 수 있습니다. 이를 통해 코드의 유연성과 확장성을 높이고 동일한 인터페이스나 부모 클래스를 통해 다양한 객체를 처리할 수 있습니다.

### 꼬리 질문 5: 자바에서 메모리 관리는 어떻게 이루어지나요?
자바는 **Heap 영역과 Stack 영역**을 사용하여 메모리를 관리합니다. 객체는 Heap에 할당되고 참조 변수는 Stack에서 관리됩니다. **Garbage Collector**가 사용되지 않는 객체를 주기적으로 회수해 메모리 누수를 방지합니다.
