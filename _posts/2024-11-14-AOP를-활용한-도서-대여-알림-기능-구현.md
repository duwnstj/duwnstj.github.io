## 개요
도서관 웹 서비스에서 사용자 경험을 개선하기 위해 알림 기능을 추가하기로 했습니다. 특히 사용자가 책을 성공적으로 대여했을 때 Gmail로 알림을 자동으로 보내는 로직을 구현하고자 했습니다. 이를 위해 Spring AOP를 활용하여 핵심 비즈니스 로직과 알림 로직을 분리함으로써 코드의 유지보수성과 가독성을 높였습니다.

## 구현 배경
알림 기능을 구현하는 과정에서 다음과 같은 요구사항을 고려했습니다.
- 책 대여 완료 시점에 자동으로 알림을 보내야한다.
- 알림 로직을 서비스 로직에 포함하지 않고 관심사를 분리해야한다.
- 비즈니스 로직 변경시 알림 로직이 영향을 받지 않도록 설계해야한다.

`Spring AOP는 특정 메서드 실행 전후에 부가적인 기능을 주입할 수 있어 이러한 요구사항을 충족하는 데 적합한 도구였습니다.


## 구현 코드
### NotificationAspect 클래스

```java
@RequiredArgsConstructor
@Aspect
@Component
@Slf4j
public class NotificationAspect {
    private final NotificationService notificationService;
    private final UserRepository userRepository;

   // 책 대여 메서드 타겟
    @Pointcut("execution(* com.example.library_management.domain.bookRental.service.BookRentalService.submitBookRental(..)) && args (bookRentalRequestDto,userDetails)")
    private void serviceLayer(BookRentalRequestDto bookRentalRequestDto, UserDetails userDetails) {
    }

    @AfterReturning(pointcut = "serviceLayer(bookRentalRequestDto,userDetails)", returning = "response", argNames = "response,bookRentalRequestDto,userDetails")
    public void sendRentalNotification(BookRentalResponseDto response,
                                       BookRentalRequestDto bookRentalRequestDto,
                                       UserDetails userDetails) {

        Long userId = bookRentalRequestDto.getUserId();
        User user = userRepository.findById(userId)
                .orElseThrow(NotFoundUserException::new);
        String email = user.getEmail();
        String message = "안녕하세요! " + user.getUserName() + "님 책 대여가 완료되었습니다! ";

        notificationService.sendEmail(email, message);

        log.info("대여 알림 전송 완료 ! :{}", email);


    }
}
```
### 코드 설명
- `@Aspect` AOP 기능을 제공하는 클래스임을 선언합니다. 특정 메서드 실행 시 부가 로직을 주입할 수 있습니다.
- `@Component` 애노테이션을 사용해 빈으로 등록해줍니다.
- `@Pointcut`: 알림 로직을 실행시킬 대상을 설정합니다. 여기서는 submitBookRental 메서드가 타겟입니다.
- `@AfterReturning`을 사용해 메서드가 실행 된 후 성공을 하게 되면 알림을 보내는 로직을 구현했습니다. `@AfterReturning`애노테이션은 메서드가 성공했을 때에만 실행이 됩니다.
- `NotificationService`: 이메일 전송 기능을 담당하는 서비스입니다.
- `UserRepository`: 사용자 정보를 조회하여 이메일과 이름을 가져옵니다.
- **로그 출력**: 전송 성공 여부를 로깅합니다.

## AOP를 사용한 알림 기능의 장점
### 관심사의 분리
- 핵심 비즈니스 로직(책 대여)과 알림 로직을 분리하여 코드의 가독성과 유지보수성을 높였습니다.
- `submitBookRental` 메서드는 알림 로직에 대한 의존 없이 자신의 역할(책 대여 처리)에만 집중할 수 있습니다.

### 코드 중복 제거
- 알림 로직을 여러곳에서 사용할 경우 AOP를 사용해 공통화할 수 있어 중복 코드를 최소화할 수 있습니다.

### 확장성
- 알림 방식(Gmail 송신)을 변경하거나 추가할 때 핵심 비즈니스 로직을 수정할 필요가 없습니다.
