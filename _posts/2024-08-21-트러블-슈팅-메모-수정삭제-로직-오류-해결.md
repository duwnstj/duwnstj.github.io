## 문제 상황
- 강의에서 제공된 예제를 따라하며 코드를 작성하고 실행해보던 중, 메모 수정 기능을 웹 사이트에서 실행했을때 수정 로직이 실행되지 않고 아래와 같은 에러 메시지가 나타났다.

![](https://velog.velcdn.com/images/duwnstj12/post/a2e7fdad-23bc-4feb-842d-13e45ea6d7ab/image.png)
- 이 에러 메시지를 잘 확인 해보니 `Long` 타입의 파라미터 값이 전달되지 않았다는 에러였다.

## 로그를 사용해 문제 추적하기
에러가 발생한 정확한 위치를 확인하기 위해 , 코드에 `Log.info()`를 추가하여 단계적으로 확인해봤다. 

1. **Service 레벨에서 로그 확인** : 
`Log.info()` 메서드를 사용해서 Service에서 로그가 출력되는지 확인했지만, 아무런 로그도 출력되지않았다.
-> Service에서 문제가 발생한 것이 아니였다.
2. **Controller 레벨에서 로그 확인** : 
그 다음으로 Controller에서 로그를 확인했지만 , 여기도 마찬가지로 로그가 출력 되지 않았다.
-> 이는 요청(Request) 값이 아예 Controller에 전달되지 않았다는 의미이다.

## 문제 원인
Controller의 코드를 다시 살펴본 결과, `@PathVariable` 애노테이션을 사용할 때 파라미터 이름을 명시적으로 지정하지 않았다는 문제가 있었다. 아래는 오류가 있던 코드이다.

```java
@PutMapping("/memos/{id}")
    public Long updateMemo(@PathVariable Long id, @RequestBody MemoRequestDto requestDto) {
        log.info("id" + id);
        log.info("requestDto" + requestDto);

        return memoService.updateMemo(id, requestDto);

    }
   
```text
- `@PathVariable`을 사용할때는 반드시 경로 변수의 이름을 지정해야하는데 , 이를 생략했기 때문에 Controller에서 요청을 제대로 처리하지 못했던 것이다.

## 해결 방법
`@PathVariable` 애노테이션을 올바르게 사용하기 위해서 아래와 같이 파라미터 이름을 명시해 주는 방식으로 코드를 수정했다.

```java
 @PutMapping("/memos/{id}")
    public Long updateMemo(@PathVariable("id") Long id, @RequestBody MemoRequestDto requestDto) {
        log.info("id" + id);
        log.info("requestDto" + requestDto);

        return memoService.updateMemo(id, requestDto);

    }
```text
- `@PathVariable("id")`처럼 경로 변수의 이름을 명시적으로 지정해야 Controller에서 해당 값을 제대로 받을 수 있다. 

## 결과 
코드를 수정한 후 , 메모 수정 및 삭제 기능이 정상적으로 작동했으며, 로그도 예상대로 출력되었다.
