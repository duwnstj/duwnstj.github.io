## 문제 설명
- 재료의 개수 N, 갑옷을 만드는데 필요한 수 M,그리고 N개의 재료 고유번호를 입력받습니다.
- 두 재료의 고유번호를 더해 M이 되는 경우의 수를 구하는 문제입니다. 

## 풀이 접근
이 문제는 투 포인터(Two Pointers) 알고리즘을 활용하면 효율적으로 해결할 수 있습니다.
- 입력 배열을 **오름차순으로 정렬**한 후, 배열의 양 끝에서 시작하여 조건을 만족하는 쌍을 찾습니다.
- 시간 복잡도는 𝑂(𝑁log⁡𝑁)입니다. 이는 배열 정렬에 𝑂(𝑁log𝑁)이 걸리고, 투 포인터 탐색은 𝑂(𝑁)으로 처리되기 때문입니다.

***

## 문제 풀이 코드
```java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.Arrays;
import java.util.StringTokenizer;

public class Main {
    public static void main(String[] args) throws IOException {
       /**
 * 1. 입력: N, M, 고유번호 배열 A
 * 2. 배열 A를 오름차순 정렬
 * 3. 시작 포인터 i=0, 끝 포인터 j=N-1로 초기화
 * 4. while(i < j):
 *   합 < M: 시작 포인터 증가 (i++)
 *   합 > M: 끝 포인터 감소 (j--)
 *   합 == M: 카운트 증가, 두 포인터 모두 이동 (i++, j--)
 * 5. 결과 출력
 */

        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        
        // N,M 입력
        int N = Integer.parseInt(br.readLine());
        int M = Integer.parseInt(br.readLine());
        
        // 재료 배열 입력 및 정렬
        StringTokenizer st = new StringTokenizer(br.readLine());
        int[] A = new int[N];
        for (int i = 0; i < N; i++) {
            A[i] = Integer.parseInt(st.nextToken());
        }
        Arrays.sort(A);
        
        // 투 포인터 탐색
        int count = 0; // 가능한 경우의 수 카운트
        int i = 0; // 시작 포인터
        int j = N - 1; // 끝 포인터
        while (i < j) {
            if (A[i] + A[j] < M) i++; 
            else if (A[i] + A[j] > M) j--;
            else { 
                count++;
                i++;
                j--;
            }
        }
        // 결과 출력
        System.out.println(count);

    }
}

```
***

## 시간 복잡도 분석
- 배열 정렬 : $$O(NlogN)$$ 
    - 배열을 정렬하는데 필요한 시간입니다.
- 투 포인터 탐색 : $$O(N)$$
    - 정렬된 배열을 한번 순회하면서 조건을 탐색합니다.
    
## 전체 시간복잡도 
$$O(NlogN)+O(N)=O(NlogN)$$
여기서 $$O(NlogN)$$이 더 오래걸리므로 최종 시간 복잡도는 $$O(NlogN)$$입니다.
