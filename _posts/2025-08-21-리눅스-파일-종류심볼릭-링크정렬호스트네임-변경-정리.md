리눅스 환경을 다루다 보면 파일 관리와 디렉터리 구조, 심볼릭 링크, 정렬, 호스트네임 변경 같은 기초 작업이 자주 등장합니다. 이번 글에서는 **실무에서 꼭 알아야 할 파일 관련 개념과 명령어**를 정리했습니다.

---

## 1. Linux에서 모든 것은 파일이다

리눅스에서는 **모든 것을 파일로 취급**합니다.

텍스트, 디렉터리, 디스크 장치, 네트워크 세션까지 모두 파일 형태로 관리됩니다.

👉 파일 타입을 확인하려면 `file` 명령을 사용합니다.

```bash
file /etc/passwd

```text
출력 예시:

```bash
/etc/passwd: ASCII text

```text
---

## 2. 파일 타입 구분 🔎

리눅스 파일은 **맨 앞 문자**로 타입을 구분합니다.

| 구분 | 의미 | 예시 |
| --- | --- | --- |
| `-` | 일반 파일(Regular file) | 텍스트, 실행 파일 등 |
| `d` | 디렉터리 | `/home`, `/etc` |
| `c` | 문자 장치(Character device) | 키보드, 터미널 |
| `b` | 블록 장치(Block device) | HDD, SSD, USB |
| `l` | 심볼릭 링크(Link file) | 바로가기 개념 |
| `s` | 소켓(Socket) | 프로세스 간 통신 |
| `p` | 파이프(Pipe) | 프로세스 연결 통신 |

👉 `ls -l` 로 확인 시 첫 번째 문자가 바로 파일 타입입니다.

---

## 3. 디렉터리 생성 팁 🏗️

- 단일 디렉터리 생성:

```bash
mkdir testdir

```text
- 상위 경로 포함 전체 생성:

```bash
mkdir -p /tmp/dev/project

```text
💡 `-p` 옵션을 쓰면 상위 폴더가 없을 때도 자동으로 생성합니다.

---

## 4. 심볼릭 링크(Symlink) 🔗

윈도우의 “바로가기”와 비슷한 개념입니다.

### 생성 방법

```bash
ln -s 원본파일 링크이름

```text
예시:

```bash
ln -s /var/tmp/ myTestLink

```text
### 결과

![](https://velog.velcdn.com/images/duwnstj12/post/655d2a60-3ac0-4900-ad8e-105f3b60510b/image.png)

- `ls` 명령어로 확인 시 링크 이름이 파랗게 표시됩니다.
- `/var/tmp/` 경로로 연결된 링크입니다.

### Dead link 문제

원본 파일이 삭제되면 링크가 깨져 **Dead link**가 됩니다.

👉 해결 방법: 원본을 다시 만들거나 링크를 재생성.

### 링크 삭제

```bash
rm myTestLink

```text
👉 링크 삭제는 원본 파일에 영향을 주지 않습니다.

### 디렉토리 링크 삭제

```bash
rm -r myTestLink

```text
- `r` 옵션을 붙이면 디렉터리 링크도 삭제 가능.

### 결과

![](https://velog.velcdn.com/images/duwnstj12/post/1cd79e9d-b5ac-4d0a-b9b1-dfd738c81d95/image.png)

- 전에 만든 링크가 사라진 것을 확인할 수 있습니다.

## 5. ls 정렬 옵션 활용 📑

- 시간순 정렬:

```bash
ls -lt

```text
- 오래된 순(역순) 정렬:

```bash
ls -ltr

```text
예시(`/etc` 디렉터리):

```bash
ls -ltr /etc

```text
출력 예시:

```text
-rw-r--r--  1 root root   296 Jul 20  2023 hosts
-rw-r--r--  1 root root   418 Jul 21  2023 hostname

```text

## 6. 호스트네임 변경하기 🖥️

리눅스에서 호스트네임은 서버 식별에 중요한 역할을 합니다.

호스트네임 변경에는 세 가지 방식이 있습니다.

### ① 임시 변경 (로그아웃 시 원복)

```bash
hostname localhost

```text
- 즉시 반영되지만, **로그아웃/재로그인 시 원래 이름으로 돌아옵니다.**

---

### ② 영구 변경 (구방식)

```bash
sudo vim /etc/hostname

```text
- 원하는 호스트네임으로 수정 후 저장.
- **주의:** 파일 수정만으로는 적용되지 않습니다.
- 반드시 다음 명령으로 커널에 즉시 반영해야 합니다:

```bash
sudo hostname localhost

```text
- 그 후 로그아웃/재로그인하거나 재부팅 시 정상 적용됩니다.

---

### ③ 영구 변경 (신방식, 권장)

```bash
sudo hostnamectl set-hostname localhost

```text
- systemd 기반 환경에서 권장되는 방식.
- `exit` → 다시 `ssh 접속`하거나 재부팅 후 적용 확인.

### 결과

![](https://velog.velcdn.com/images/duwnstj12/post/f9718994-bda5-4c85-b479-b1a37c28238b/image.png)

- `@` 옆 프롬프트의 이름이 변경되면 정상 적용된 것입니다.

## 7. history 명령 활용 ⏳

리눅스에서는 사용자가 입력한 명령어 이력이 저장됩니다.

- 일반 사용자 명령어 확인:

```bash
history

```text
- root 사용자 명령어 확인:

```bash
sudo -i or sudo su
history

```text
👉 `history` 이력은 사용자 계정별로 따로 관리됩니다.

👉 실무에서는 **작업 기록 확인·복구·보안 감사** 용도로도 많이 사용됩니다.
