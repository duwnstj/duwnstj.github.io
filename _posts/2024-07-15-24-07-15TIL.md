# 중요한 리눅스 명령어
`PWD` : pwd를 git bash에서 입력을 하면 현재 폴더의 위치(경로)를 알 수 있다. 

`ls` : ls 명령어는 현재 내가 있는 폴더 안에 있는 파일들이나 폴더들을 보여준다.

`ls -a` : ls -a 명령어를 치게 되면 .파일들을 포함하여 모든 파일들을 다 보여준다. 이 `.파일`이 바로 숨겨진 파일이라고 한다.

`cd` : ls를 먼저 쳐서 나오는 폴더들에 cd 명령어 이후 이동하고 싶은 폴더 예를 들어 `cd desktop` 이런 식으로 명령어를 친다면? desktop 폴더로 이동하게 된다.
`cd ..`은 현재 위치에서 한단계 상위 위치로 이동하는 cd 명령어이다.
**만약 두단계 상위 위치로 이동하고 싶다면?**
`cd ../..`을 하면 된다. 이 의미가 상위 경로로 이동하고 한번더 이동이라는 뜻이다.
** 그럼 상위 위치에 어떤폴더들이 있는지 알 수 있는 방법은?**
바로 위에서 작성한 `pwd`명령어를 통해 알 수 있다.
예를 들어 pwd 명령어를 입력하면 /c/user/users/desktop 이런식으로 경로가 나오게 되는데 /를 기준으로 경로가 나뉘어있고 이 상태에서 cd .. 명령어를 작성해서 pwd로 경로를 확인하면 /c/user/users로 나오는걸 확인 할 수 있다.

`mkdir` : `mkdir` 명령어는 폴더(디렉토리)를 생성하는 명령어이다. 
바탕화면에서 새폴더를 눌러 폴더 생성하는 것과 같은 개념이다.

`touch` : touch text.txt를 치게 되면 text파일을 만들 수 있다. 

# Git을 사용 하는 이유
우리가 과제를 할 때를 생각해보면 한번에 작성하기는 힘들다. 수정해서 저장하고 이렇게 만들다보면 파일이 많아지는데 각 파일에 어떤점이 어떻게 변했는지 알 수가 없어서 어떤게 최종본인지도 잘 알 수 가 없다. 이런점들을 Git으로 해결 할 수가 있다.

## Git&Github이란?
Git은 코드 변경점을 기록 하는 것이라 말할 수 있다. 
코드에 변경된 사항들이 있다면 commit 메시지를 통해 변경 된 부분들을 쉽게 확인 할 수 있다. 그럼 Github이란 뭘까? git과 github은 다른 개념이다. 
github은 cloud를 생각하면 쉽다. cloud 저장소에 백업해야하는 파일들을 저장하듯이 github도 이런 온라인 원격 저장소라고 생각하면 된다.
Github에서는 git으로 저장한 파일들을 `온라인 백업,공유,협업`을 할 수 있게 해줘서 Github으로 협업이 가능하다.

## Git 명령어
`git init` : 이 명령어를 사용하면 폴더 안에 .git이라는 폴더가 생성이 된다. 이 폴더가 생성이 되면 git을 사용 할 수 있게 된다.이 명령어를 칠 때 중요한 점은 정확한 프로젝트 폴더(경로)에서 입력을 해야한다는 점이다. 
**만약 정확한 프로젝트 폴더를 입력하지않으면 공유하고 싶지 않은 파일들도 같이 공유가 될 수 있기 때문에 조심해야한다.**

`git add` : git add 명령어를 통해 저장할 파일을 지정한다.

`git commit`: git add 명령어를 통해서 지정한 파일을 `git commit -m "커밋 메시지"`를 입력하여 저장을 하게 된다.
이 커밋 메시지에는 변경한 사항을 쉽게 알아볼 수 있게 자세히 쓰는 것이 좋다.

`git status` : git status 명령어를 사용하면 현재 변경된 파일들의 저장 상태를 볼 수 있다. 저장되지 않은 파일은 빨간색으로 나타내준다.

**만약 저장한 파일들이 10개 이상 된다고 한다면?** `git add .` 명령어를 사용해 현재 변경 사항이있는 파일들을 전부 지정할 수 있다.
파일들을 전부 지정한 뒤 commit명령어를 통해 git에 저장을 하면 된다.

`git log` : 이 명령어를 입력하면 이 폴더에서 작성한 이전 커밋들이 전부 나오게 된다. `git diff` 명령어를 통해 이 코드에서 어떤 변경이 있었는지 확인 할 수 있고, `git reset`명령어를 통해서 이전 커밋 기록으로 되돌아갈 수는 있지만, 
**이 방법은 프로젝트를 협업할 때 사용하지 않는 것이 좋다. 다른 사람들의 커밋 메시지도 꼬이기 때문이다.**

이렇게 git에 저장한 코드들을 이제 원격 저장소인 Github에 저장할 수 있다.
