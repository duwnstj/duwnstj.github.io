# 🧠 Workspace-Scoped Rules (블로그 프로젝트 전용)

### 1. 다이어그램(Mermaid) 및 퀄리티 우선주의 (No Image Capture)
- **이미지 캡처 전면 금지**: 블로그 원고 작성 시 화면 캡처 이미지를 관리하거나 요구하는 과정을 완전히 배제한다. 단순 스크린샷은 아키텍처 블로그의 본질적 가치를 높이지 못한다.
- **직관적인 시스템 다이어그램**: 이미지 캡처 대신, 복잡한 인프라 구조나 트러블슈팅 파이프라인은 반드시 **Mermaid 다이어그램**을 사용하여 구조적으로 시각화한다.

### 2. 서사(Storytelling) 중심의 편집 (Aha-Moment 보존)
- 단순한 코드 나열이나 뻔한 성공담은 엄격히 배제한다.
- **CRITICAL**: "어떤 에러가 발생했는가 ➔ 왜 발생했는가 ➔ 어떤 비유를 통해 아키텍처를 통찰했는가 ➔ 어떻게 해결했는가"라는 본질적인 사고의 흐름과 텍스트 퀄리티 자체를 유지한다.
- 원고 내에 사용자의 깨달음(Aha-Moment, 예: "아! 이게 이런 구조였구나!")이나 대화형 티키타카 서사가 포함되어 있다면, 이를 절대 훼손하지 않고 날것 그대로 렌더링에 반영한다.
