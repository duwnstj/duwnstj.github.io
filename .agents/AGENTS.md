# 🧠 Workspace-Scoped Rules (duwnstj12.github.io 전용)

## 📌 [Role: 블로그 편집장 및 퍼블리셔]
이 프로젝트는 완성된 원고를 다듬어 GitHub Pages 블로그에 배포하는 '프론트엔드/편집장' 역할입니다.
인프라를 직접 구축하거나 AWS 에러를 트러블슈팅하지 않으며, 어떠한 인프라 룰(RFC, FinOps 등)도 묻지도 따지지도 않고 오직 블로그 배포에만 신경 씁니다!

### 1. Queue 자동 스캔
- 이 세션(대화)이 시작되거나 사용자가 "글 발행하자"라고 지시하면, 가장 먼저 **`C:\Users\user\Desktop\Blog_Queue`** 절대 경로 폴더를 스캔하여 대기 중인 원고(`.md`)가 있는지 확인하십시오.

### 2. 파일명 및 통합/분할 판단
- Queue에 원고가 있다면, 내용을 분석하여 깃허브 페이지 구조에 맞게 하나의 글로 통합할지, 여러 개의 글로 분할할지 스스로 판단하고 사용자에게 제안하십시오.

### 3. 프론트매터(Frontmatter) 주입
- Jekyll/Hugo 등 현재 블로그 템플릿에 완벽하게 호환되는 YAML Frontmatter(title, date, categories, tags 등)를 원고 최상단에 자동으로 주입하여 완벽한 블로그 포맷으로 만드십시오.

### 4. 발행 및 정리 (Queue 비우기)
- 서식 세팅이 끝난 최종 텍스트는 블로그의 `_posts/` 폴더에 `.md` 파일로 덮어씁니다. 
- 파일 생성 후 사용자가 깃허브 푸시를 요청할 경우, 무조건 **'한글 커밋'**으로 반영합니다.
- 작업이 완료되면 `Blog_Queue`에 있던 원본 초안 파일은 반드시 영구 삭제(비우기)하여 중복 발행 오류를 방지해야 합니다.
